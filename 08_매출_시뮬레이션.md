## 1. 시나리오별 월 매출 (회사 수취 기준)

`MAU × 구독전환율 × 구독단가`

| 시나리오 | 계산 | 월 구독 매출 |
| --- | --- | ---: |
| 보수적 | 3,000 × 2% × 12,000 | **720,000원** |
| 기본 | 10,000 × 4% × 14,000 | **5,600,000원** |
| 낙관적 | 30,000 × 7% × 16,000 | **33,600,000원** |

### 1-B. 판매 수수료 매출
`MAU × 구매발생률 × 평균결제액 × 30%`

| 시나리오 | 계산 | 월 판매 수수료 |

| 보수적 | 3,000 × 1% × 2,000 × 30% | **18,000원** |

| 낙관적 | 30,000 × 5% × 5,000 × 30% | **2,250,000원** |

### 1-C. 후원 수수료 매출
`MAU × 후원율 × 평균후원액 × 월빈도 × 15%`

| 시나리오 | 계산 | 월 후원 수수료 |

| 낙관적 | 30,000 × 5% × 4,000 × 1.5 × 15% | **1,350,000원** |

## 3. 핵심 인사이트 (숫자가 말하는 것)

1. **매출의 90%+는 '창작 도구 구독'에서 나온다.**
   → 판매·후원은 금액 자체는 작다. **회사의 생존은 구독/크레딧이 좌우**한다. (앞서 "마켓이 메인 사업이 되긴 어렵다"고 한 진단이 숫자로 확인됨.)

2. **판매·후원의 진짜 가치는 '직접 매출'이 아니라 '리텐션·유입'.**
   → 후원받은 창작자는 떠나지 않고 더 만든다 → 구독 전환·트래픽 증가로 **간접적으로 구독 매출을 키운다.** 후원/판매는 *생태계 윤활유*로 평가해야 한다.

3. **변수 민감도: MAU > 구독전환율 > 구독단가 순으로 매출에 큰 영향.**
   → 결국 **"사용자를 얼마나 모으냐(MAU)"** 가 전부. 마케팅·바이럴(표지 자랑 등)이 1순위 과제.

4. **손익분기 감각**: 기본 시나리오 월 600만 원대 매출은 소규모 팀(2~3인) 인건비·서버·AI API 비용을 겨우 감당하는 수준. **흑자는 낙관 시나리오(MAU 3만)부터.** → MAU 1만은 생존선, 3만이 본궤도.

## 4. 비용 측면 주의 (매출만 보면 안 됨)

| 비용 항목 | 특성 | 비고 |
| --- | --- | --- |
| **AI 생성 API 비용** | 변동비·매출과 연동 | 회차/표지 생성마다 발생 → **크레딧 단가 < API 원가면 적자** |
| 인프라·서버 | 반고정비 | 트래픽 비례 |
| 결제대행(PG) 수수료 | 변동비 | 후원·판매 정산 시 차감 |
| 인건비 | 고정비 | 초기 최대 비용 |
| 마케팅비(CAC) | 변동비 | MAU 확보 핵심, ROI 관리 필수 |

> ⚠️ **가장 큰 함정**: AI 생성 1건당 API 원가가 크레딧 판매가보다 비싸면, **많이 쓸수록 적자**가 난다. 크레딧 단가 설계 시 API 원가에 마진을 반드시 얹어야 한다.

## 5. 검증 우선순위 (이 시뮬레이션을 신뢰하려면)

| 순위 | 검증할 가정 | 검증 방법 |
| :-: | --- | --- |
| 1 | 구독 전환율(2~7%) | 베타 유료화 A/B 테스트 |

| 4 | 후원/판매 전환율 | 마켓 베타에서 실측 |

> **결론**: 매출 가능성은 **구독 모델이 받쳐주면 충분히 사업화 가능**. 단, ①MAU 확보 ②크레딧 단가>API원가 두 가지가 깨지면 숫자는 무의미. 후원·판매는 '메인 매출'이 아니라 '창작자를 묶어두는 장치'로 운영하는 게 맞다.
