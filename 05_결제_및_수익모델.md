# 결제 및 수익 모델

> 세 개의 매출 엔진: **① 창작 도구 매출(구독·크레딧)** + **② 마켓플레이스 수수료(C2C 거래)** + **③ 자율 후원 수수료(포스타입式 팁)**
> 핵심: 사용자가 **내 소설을 판매·후원받고**, **타인의 소설을 구매·후원**하는 양면 시장(Two-sided Market)
> 💡 후원·판매 등 **모든 창작자 수익 거래에서 플랫폼이 일정 수수료를 수취**하는 것이 회사의 핵심 매출원.

---

## 1. 수익 모델 전체 구조

| 매출원 | 설명 | 과금 대상 | 유형 |
| --- | --- | --- | --- |
| **A. 구독(Subscription)** | 월 정액으로 생성 횟수·고급기능 제공 | 창작자 | 정기결제 |
| **B. 크레딧(Credit)** | 회차/표지 생성 1건당 크레딧 차감, 부족 시 충전 | 창작자 | 종량제 |
| **C. 마켓 판매 수수료** | 작품 거래액의 일정 % 플랫폼 수취 | 판매자↔구매자 | 거래 수수료 |
| **D. 콘텐츠 구매** | 독자가 유료 작품/회차 구매 | 구매자 | 단건/묶음 결제 |
| **E. 자율 후원(Tip) 수수료** | 무료 공개작에 독자가 자율 후원 → **후원금의 일정 % 플랫폼 수취** | 후원자→창작자 | 거래 수수료 |
| **F. 부가 수익** | 프로모션 노출, 표지 고급팩, 광고(무료 유저) | 전체 | 옵션 |

| --- | --- | --- | --- | --- |

## 3. 🛒 마켓플레이스 수익 모델 (핵심)

### 3-1. 거래 방식
- **판매자(창작자)**: 작품을 `유료 판매`로 등록, **회차별 가격** 또는 **전체 묶음 가격** 설정.
- **구매자(독자)**: 미리보기(1~2화 무료) 후 회차/작품 단위로 구매.
- **결제 수단**: 플랫폼 공통 화폐 **'잉크(가칭 포인트/캐시)'** 충전 후 사용 → 정산 단순화.

### 3-2. 정산 구조 (Revenue Share)

| 구분 | 비율(예시) | 비고 |
| --- | :---: | --- |
| 창작자 정산 | **70%** | Pro 플랜은 80%로 우대(구독 전환 유인) |
| 플랫폼 수수료 | **30%** | 결제대행·서버·운영 포함 |
| 최소 출금액 | 10,000원 | 정산 주기 월 1회 |

### 3-3. 판매 가격 가이드
| 단위 | 권장 가격대 |
| --- | --- |
| 회차 단건 | 100~300원 |
| 완결 작품 묶음 | 3,000~9,900원 |
| 무료 미리보기 | 도입 1~3화 |

## 3.5. 💝 자율 후원(Tipping) 모델 — 포스타입式

**핵심 목표**: "AI 글에 왜 돈 내?"라는 하드 페이월 거부감을 우회. 무료로 읽게 한 뒤, 좋으면 **자발적으로 창작자를 응원**하게 한다. 후원은 'AI 글값'이 아니라 **'창작자의 기획·노력에 대한 응원'** 으로 프레이밍.

### 3.5-1. 작동 방식
- 작품을 **무료 공개** → 누구나 장벽 없이 읽음(바이럴·트래픽↑).
- 회차 끝·작품 하단에 **[이 작품 응원하기]** 후원 버튼 노출.
- 독자가 원하는 금액(예: 1,000 / 3,000 / 5,000 / 직접입력)을 후원.
- **후원금에서 플랫폼 수수료를 차감**하고 나머지를 창작자에게 정산.

### 3.5-2. 후원 정산 구조 (플랫폼 수수료 수취)
| 구분 | 비율(예시) | 비고 |
| --- | :---: | --- |
| 창작자 정산 | **80~90%** | 후원은 순수 응원 성격 → 판매보다 창작자 몫 높게 책정 |
| **플랫폼 수수료** | **10~20%** | 결제대행·운영비 포함, 회사 매출원 |
| 결제대행(PG) 수수료 | 별도/포함 | 수수료 내 흡수 또는 별도 고지 |
| 최소 출금액 | 10,000원 | 판매·후원 통합 정산 |

> 📌 후원 수수료(10~20%)를 판매 수수료(30%)보다 **낮게** 두는 이유: 후원은 진입장벽을 낮춰 트래픽을 키우는 미끼 역할 → 창작자 유입·활동을 늘려 결과적으로 판매·구독 매출까지 견인.

### 3.5-3. 후원 유인 장치(전환율 보완)
> 후원 전환율은 통상 읽은 사람의 1~5% → 보상으로 유인 강화.
- **후원자 전용 보상(선택)**: 다음 회차 미리보기, 비하인드 설정/관계도, 감사 메시지.
- **후원 랭킹·뱃지**: 'Top 후원자' 노출로 팬덤 형성.
- **목표 후원(크라우드펀딩식)**: "○○원 모이면 외전 연재" 같은 목표 게이지.

### 3.5-4. 후원 플로우
```
[M3 뷰어 / 작품 하단] → [응원하기 버튼]
  → [후원 금액 선택(프리셋/직접입력)] → [잉크 잔액 확인]→(부족 시)[충전]
  → 결제(간편결제) → [후원 완료 + 감사 화면] → 후원자 보상 지급(선택)
  → 창작자: [S1 대시보드 후원 수익 집계] → (수수료 차감 후) 정산
```
| 단계 | 사용자 감정/기대 |
| --- | --- |
| 무료 열람 | 😀 "부담 없이 읽는다" |
| 응원 결정 | 🥰 "좋았으니 고마움 표시" (자발성) |
| 후원 완료 | 😌 "창작자 응원했다" 만족 + 보상 획득 |
| (창작자) 수익 확인 | 🤩 "내 작품이 응원받았다" → 창작 지속 동기 |

## 4. 결제 플로우

### 4-1. 구매자(독자) 결제 플로우
```
[M2 판매페이지] →(미리보기 열람)→ '구매하기'
  → [잉크 잔액 확인] → (부족 시)[충전 모달] → 결제(카드/간편결제)
  → [구매 완료] → [M3 뷰어 열람] → [B1 내 서재 보관]
```
| 단계 | 사용자 감정/기대 |
| --- | --- |
| 미리보기 | 🤔 "재밌으면 살게" |
| 충전·결제 | 😐 "결제 간편했으면" (간편결제 필수) |
| 열람 | 🤩 "바로 읽힌다" 만족 → 다음화 연속구매 |

### 4-2. 판매자(창작자) 수익화 플로우
```
[C4 에디터 완성] → [S2 가격설정·공개등록] → [M2 마켓 노출]
  → 타 사용자 구매 발생 → [S1 대시보드 매출 집계] → [정산 출금]
```

## 5. 단계별 수익화 로드맵 (Phasing)

| 단계 | 시점 | 적용 모델 | 목표 |
| --- | --- | --- | --- |
| **Phase 1 (MVP)** | 출시 | Free + 크레딧 + 구독(Basic) | 생성 경험·결제 검증 |
| **Phase 2** | +3개월 | 마켓플레이스 오픈(판매/구매) | 양면 시장 활성화 |
| **Phase 3** | +6개월 | Pro 플랜·정산 우대·프로모션 노출 | 우수 창작자 락인 |
| **Phase 4** | +12개월 | 추천 알고리즘·IP 2차 활용(웹툰화 연계) | 생태계 확장 |

## 6. 리스크 및 정책 고려사항
- **저작권/표절**: AI 생성물의 권리 귀속, 타 작품 표절 검수 필요(등록 시 유사도 검사).
- **콘텐츠 심의**: 19금·불법 콘텐츠 필터링, 판매 전 검수/신고 시스템.
- **정산 투명성**: 판매·수수료 내역 실시간 공개로 창작자 신뢰 확보.
- **부정거래 방지**: 자전거래(셀프구매)·환불 어뷰징 모니터링.
- **AI 생성 고지**: 작품에 'AI 생성' 라벨 표기로 소비자 보호.
