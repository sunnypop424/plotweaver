# 플롯위버 — 화면 디자인 프롬프트 모음 (C2 → C3 → C4 → C5 → L)

> **용도**: Claude 디자인에 **한 화면씩** 붙여넣어 고충실도 화면을 생성하기 위한 프롬프트 모음.
> **순서**: 아래 큰 분류를 **위에서부터 차례대로** 진행한다(C1 설정 위저드는 이미 완료).
> **항상 함께 첨부**: `DESIGN.md`(토큰) + `화면기획_디자인브리프.md`(해당 화면 섹션).

---

## 0. 사용법

1. 아래 **`공통 규칙`** 을 먼저 1회 읽히거나, 각 화면 프롬프트 맨 앞에 같이 붙인다.
2. **한 번에 한 화면**만. 위 순서(C2→C3→C4→C5→L)대로.
3. 첨부: `DESIGN.md` + `화면기획_디자인브리프.md`(가능하면 해당 §만 발췌).
4. 직전 화면에서 만든 **공통 컴포넌트를 재사용**하라고 명시(일관성 ↑).

---

## 공통 규칙 (모든 화면 프롬프트에 적용)

```
[공통 규칙 — 모든 화면 공통]
- 토큰 출처는 DESIGN.md(Laftel) 단일 출처. 색·폰트·간격·라운드를 임의값으로 쓰지 말 것.
  · primary #816BFF / primary-hover #6E58FF / wash #F0EDFF / hover-wash #D9D3FF
  · text #121212 / text-2 #505050 / muted #8A8A8A / border #EEEEEE / error #F16361 / toast #242537
  · 폰트 Pretendard / 라운드: 버튼·입력 4px, 카드 8px, 알약 9999px
  · 모션: 색상 전환 0.4s, opacity/transform 0.2s ease, 스켈레톤 1.5s linear
- 산출물: React + TypeScript + Tailwind 단일 화면 컴포넌트(인터랙티브, 더미 데이터 내장).
  공통 컴포넌트(예: HybridSelect, Stepper, Toast)는 별도 파일로 분리해 재사용.
- 반응형 브레이크포인트: 1280 / 1024 / 768 / 480. 모바일(<768)은 1단 + 하단 고정 CTA 바.
- ★하이브리드 입력 원칙(중요): 모든 Select에는 "+ 직접입력" 토글을 둬 자유 텍스트로 전환 가능해야 한다.
  (배경시대·관계라벨·외형·성격·표지스타일·색감 등 전부)
- 아이콘은 Lucide(또는 인라인 SVG, currentColor) 사용. 이모지 지양(DESIGN.md 정책).
  단 AI 생성 액션의 ✦(sparkle)만 브랜드 악센트로 허용.
- 상태(빈/로딩/에러/성공/비활성)를 반드시 모두 구현.
- 먼저 DESIGN.md 토큰을 어떻게 적용할지 1~2줄로 말한 뒤 컴포넌트 코드를 출력.
```

# 1. C2 — 관계도 빌더 (★가장 까다로움)

> 위저드 ③단계. **상단 바 + 스텝 인디케이터(현재 ③)는 C1과 동일 chrome 재사용.** 핵심은 좌측 그래프 캔버스 + 우측 편집 패널.

```
[공통 규칙]을 따른다. 첨부한 DESIGN.md·브리프(§2-4)를 따라 화면 하나를 고충실도로 디자인해줘.

[대상 화면] §2-4 관계도 빌더 (C2) — 인물 관계와 회차별 변화를 시각화. ★서비스 차별화 화면.
C1 위저드의 상단 바("← 뒤로 / 새 작품 만들기 / 임시저장")와 4스텝 인디케이터를 그대로 재사용하되, 현재 단계는 ③관계도 활성으로.

[레이아웃 — Desktop 2단]
- 좌측(넓게): 관계 그래프 캔버스
  · 노드 = 등장인물(원형 칩, 이름). 드래그로 위치 이동.
  · 엣지 = 관계(곡선 + 중앙에 관계 라벨, 방향 화살표). 클릭 시 하이라이트 → 우측 패널에서 편집.
  · 좌하단 [+ 관계 추가], 우상단 [자동 추천].
- 우측 패널: 관계 편집
  · From ▼(인물) / To ▼(인물)
  · 관계 ▼(연인/원수/동료/사제/형제/동맹/배신자 …) + "+ 직접입력"
  · 방향 (•)양방향 ( )일방향  (라디오)
  · ─ 관계 변화(타임라인) ─  "5화: 동료 → 연인"  "12화: 연인 → 원수"  [+ 변화 추가]
     (각 항목: 회차 숫자입력 + 이전라벨 → 새라벨)
- 하단 바: [← 이전]            [ ✦ 1회차 생성하기 ]  (주 CTA, 가장 강조)

[핵심 컴포넌트 — 재사용 가능하게 분리]
1) GraphCanvas: SVG 기반. 노드 좌표는 state로 관리(드래그 시 갱신). 엣지는 두 노드 사이 곡선(quadratic) + 라벨 + 화살촉. 노드/엣지 선택 상태.
2) RelationEditPanel: From/To/관계(HybridSelect 재사용)/방향.
3) TimelineEditor: 회차별 관계 변화 행 추가/삭제(회차 오름차순 정렬).

[더미 데이터]
- 노드 3: 카엘(주인공), 리나, 제로드(악역)
- 엣지 2: 카엘─리나 "연인"(양방향) / 카엘─제로드 "원수"(양방향)
- 카엘─리나 타임라인: 5화 동료→연인, 12화 연인→원수

[상태 — 모두 구현]
- 인물 2명 미만: 캔버스에 "인물을 2명 이상 추가하면 관계를 만들 수 있어요" 가이드.
- 빈 캔버스(관계 0): 점선 안내 + [+ 관계 추가] 강조.
- 자동 추천 클릭: 로딩(스피너) → 추천 관계 채움 토스트.
- 엣지 선택: 해당 엣지 강조 + 우측 패널에 값 로드.
- 모바일(<768): 그래프 캔버스 위 / 편집 패널 아래로 세로 스택. 하단 고정 [1회차 생성] 바.

[디자인 원칙]
- 캔버스는 차분한 캔버스 배경(#F7F7F7) 위 흰 노드, 관계 라벨은 wash(#F0EDFF)/primary 알약.
- "관계가 시각화되네"라는 신뢰감(브리프 §0-5). 복잡해 보이지 않게 여백 충분히.
```

# 2. C3 — 생성 로딩

> 이탈이 가장 큰 구간. 불안 해소가 전부.

```
[공통 규칙]을 따른다. 첨부한 DESIGN.md·브리프(§2-5)를 따라 화면 하나를 디자인해줘.

[대상 화면] §2-5 생성 로딩 (C3) — AI가 1회차를 집필하는 동안의 대기 화면.

[레이아웃 — 화면 중앙 정렬, 단일 컬럼]
- 상단: ✦ "AI가 1회차를 집필 중입니다..." (title-m, 차분한 애니메이션)
- 진행률 바: [██████████░░░░] 62%  + "예상 20초" (진행률은 더미로 자동 증가 시뮬레이션)
- 입력 요약 카드: "중세 유럽 · 회귀/복수 · 카엘(주인공) / 목표: 복수 / 결말: 복수 완성"

[핵심 컴포넌트]
1) ProgressBar(value): wash 트랙 + primary 채움, 부드러운 전환.

- 실패(에러): 빨강(#F16361) 아이콘 + "생성에 실패했어요. 크레딧은 차감되지 않았어요." + [다시 시도] (secondary).
  ※ "크레딧 미차감" 안심 문구 필수(12 무결성 규칙).
- 토글 버튼 3개(진행/완료/실패)로 상태를 데모 전환 가능하게.

[디자인 원칙]
- 과한 연출 금지, 차분·집중. 진행률·예상시간·설정요약으로 "잘 되고 있다"는 신뢰.
```

# 3. C4 — 에디터 / 뷰어 (★Aha 화면)

> "내 설정이 소설이 됐다"는 핵심 감동 + 편집(창작 기여, 문서 20).

```
[공통 규칙]을 따른다. 첨부한 DESIGN.md·브리프(§2-6)를 따라 화면 하나를 디자인해줘.

[대상 화면] §2-6 에디터/뷰어 (C4) — 생성된 회차 본문을 읽고 편집/이어쓰기. ★핵심 Aha.

[레이아웃 — Desktop 2단(좌측 회차목록 + 본문)]
- 상단 바: [← 작업실]  「회귀한 검, 황혼을 베다」      [표지 만들기] [판매등록]
- 좌측 사이드: 회차 목록(≡)
  · ●1화 회귀  / 2화 …  / 3화 …  / [+ 다음 회차 생성]
  · 현재 회차 강조.
- 본문 영역:
  · 제목 "1화. 회귀"
  · 본문(읽기 모드: 편안한 리더 타이포 / 편집 모드: 인라인 편집)
  · 우상단 작은 토글 [읽기 ↔ 편집]
  · 본문 하단 액션: [✎ 부분 수정] [↻ 이 회차 재생성]   [ 다음 회차 생성 → ]
- 기여 게이지(문서 20): 본문 영역 상단/사이드에 "직접 편집 비중" 진행 게이지 + "판매하려면 직접 편집이 필요해요" 안내(편집할수록 채워짐).

[핵심 컴포넌트 — 재사용]
1) ChapterList: 회차 항목(번호·제목·상태), 모바일은 드로어(햄버거).
2) ContentReader / ContentEditor: 읽기 타이포(가독형) ↔ 편집(textarea/contenteditable) 토글.
3) ContributionGauge(percent): 편집 기여도 시각화(낮음→"AI 생성" / 충분→"AI 보조").
4) ChapterActions: 부분수정/재생성/다음생성.

[더미 데이터]
- 작품: 「회귀한 검, 황혼을 베다」 / 회차 1~3, 현재 1화.
- 1화 본문(샘플, 3인칭): "눈을 떴을 때, 카엘은 10년 전으로 돌아와 있었다. 처형대의 차가운 감촉 대신…" (4~6문단 더미)
- 기여 게이지: 18% (아직 "AI 생성" 라벨)

[상태 — 모두 구현]
- 읽기/편집 모드 토글.
- 재생성: 본문 자리 스켈레톤/로딩.
- 저장: "저장됨" 토스트(편집 후).
- 마지막 회차: [다음 회차 생성 →] 강조 / 생성 중 비활성.
- 모바일(<768): 회차 목록 드로어, 하단 고정 [다음 회차 생성] 바.

[디자인 원칙]
- 본문은 몰입·집중(넉넉한 행간, 적당한 폭 ~680px). 도구 UI는 본문을 방해하지 않게 절제.
- 편집을 "쉽고 의미있게"(문서 20 §2) — 게이지로 편집을 자연스럽게 유도.
```

```
[대상 화면] §2-7 표지 생성 (C5) — 설정 기반 책 표지 자동 생성·선택·확정.

[상태 — 모두 구현]

[디자인 원칙]
```

# 5. L — 랜딩

> 비로그인 첫 접점. "이거 진짜 되네?" → 가입 전환.

```
[공통 규칙]을 따른다. 첨부한 DESIGN.md·브리프(§2-1)를 따라 화면 하나를 디자인해줘.

[대상 화면] §2-1 랜딩 (L) — 서비스 소개 + 가입 전환. 비로그인 상태.

[레이아웃 — 세로 섹션 스택]
- 글로벌 네비(sticky): 플롯위버(로고)  [창작] [마켓]        [로그인] [무료로 시작]
```
