# 플롯위버(PlotWeaver) — 서비스 컨셉 · 페르소나 · MVP · 유저플로우

> AI 기반 웹소설 생성기 / 작성: 수석 PM / 최종수정: 2026-06-26

---

## 1. 서비스 컨셉 및 핵심 가치 제안 (Value Proposition)

### 1-1. 한 줄 정의
> **"설정만 넣으면, AI가 회차별 웹소설과 책 표지까지 완성해 주는 창작 스튜디오"**

### 1-2. 서비스 컨셉
플롯위버는 '글을 잘 쓰는 능력'이 아니라 '설정을 구조화하는 능력'만으로 웹소설을 완성할 수 있게 해주는 SaaS형 창작 도구입니다. 사용자는 **배경 시대, 등장인물, 장르, 사건 흐름, 인물 관계, 회차 수** 등 핵심 설정만 입력하면, AI가 일관성 있는 회차별 본문을 생성하고 어울리는 책 표지 이미지까지 만들어 줍니다.

### 1-3. 핵심 가치 제안 (Value Proposition Canvas)

| 구분 | 내용 |
| --- | --- |
| **고객의 일(Job-to-be-done)** | 머릿속 이야기/설정을 '읽을 수 있는 완성된 소설'로 만들고 싶다 |
| **Pain (불편)** | 문장력 부족, 연재 분량의 압박, 설정 일관성 붕괴, 막막한 백지 |
| **Gain (기대)** | 빠른 완성, 일관된 캐릭터, 그럴듯한 표지, 자랑/연재할 결과물 |
| **Pain Reliever** | 설정 기반 자동 본문 생성 · 캐릭터/관계 일관성 유지 엔진 |
| **Gain Creator** | 회차별 원클릭 생성 · AI 표지 · 관계도 시각화 · 즉시 공유 |

### 1-4. 차별화 포인트
1. **관계도 기반 일관성** — 인물 관계도와 '관계 변화 타임라인'을 데이터로 관리해, 회차가 진행돼도 캐릭터가 무너지지 않음.
2. **설정 → 본문 → 표지 원스톱** — 글과 이미지를 한 흐름에서 해결.
3. **Select + 직접입력 하이브리드** — 초보자는 추천값으로 빠르게, 고수는 직접입력으로 디테일하게.

## 2. 타겟 페르소나 3종 (인구통계 · 동기 · Pain Point)

| 항목 | 페르소나 A — '설정덕후 입문 창작자' | 페르소나 B — '연재 부담 현직 웹소설가' | 페르소나 C — '취미·힐링 소비형 독자' |
| --- | --- | --- | --- |
| **이름/나이** | 김민준 / 17세 | 이서연 / 29세 | 박지훈 / 38세 |
| **직업** | 고등학생 | 웹소설 연재 작가(겸업) | 회사원(사무직) |
| **라이프스타일** | 웹툰·웹소설 다독, 커뮤니티 활동, 모바일 중심 | 본업+연재 병행, 마감에 쫓김, 노트북 작업 | 출퇴근 지하철 독서, 주말 취미 글쓰기 욕구 |
| **이용 동기** | "내 설정으로 진짜 소설 만들어 자랑하고 싶다" | "초안/회차 분량을 빨리 뽑아 마감을 줄이고 싶다" | "직접 못 쓰지만 내 취향 이야기를 읽고 싶다" |
| **가장 큰 Pain Point** | 문장력이 부족하고 끝까지 못 씀(백지의 공포) | 매일 연재 분량 압박, 아이디어 고갈, 번아웃 | 시간·재능 부족, 완성된 결과물을 못 만듦 |
| **핵심 니즈** | 쉬운 입력 + 그럴듯한 완성본 | 빠른 초안 + 설정 일관성 | 최소 입력 + 즉시 읽을 결과물 |
| **결제 성향** | 낮음(무료~소액), 부모 결제 의존 | 높음(생산성 도구엔 지불) | 중간(구독 거부감 적음) |

## 3. 핵심 기능(MVP) 정의서 — BRD/PRD 초안

| 우선순위 | 기능명 | 기능 설명 | 구현 난이도 | 비즈니스 임팩트 |
| :---: | --- | --- | :---: | :---: |
| ⭐1 | **설정 입력 위저드** | 배경시대/인물/장르/흐름/관계/회차를 단계별로 입력(Select+직접입력) | 중 | 상 |
| ⭐2 | **회차별 본문 자동 생성** | 입력 설정 기반으로 회차 단위 소설 본문을 AI가 생성 | 상 | 상 |
| ⭐3 | **AI 책 표지 생성** | 제목·장르·분위기 기반으로 표지 이미지 자동 생성 | 중 | 상 |
| 4 | 인물 관계도 빌더 | 등장인물 간 관계를 노드-엣지로 시각화, 회차별 변화 설정 | 상 | 중 |
| 5 | 작품 보관함(라이브러리) | 생성한 작품/회차 저장·재편집·이어쓰기 | 하 | 중 |
| 6 | 회차 재생성·부분 수정 | 마음에 안 드는 회차/문단 다시 생성, 톤 조정 | 중 | 중 |
| 7 | 공유·내보내기 | 링크 공유, TXT/PDF/이미지 내보내기 | 하 | 중 |
| 8 | 회원가입/온보딩 | 소셜 로그인, 첫 작품 가이드 투어 | 하 | 상 |
| 9 | 결제·크레딧 시스템 | 구독/크레딧 차감, 생성 횟수 관리 | 중 | 상 |
| 10 | 커뮤니티·공개 갤러리 | 다른 사용자의 공개 작품/표지 탐색 | 중 | 하 |

> **🎯 고객이 가장 먼저 경험해야 할 핵심 3종 (Aha-moment 설계)**
> 1. **설정 입력 위저드** — 막막함 없이 따라만 하면 설정이 완성된다.
> 2. **회차별 본문 자동 생성** — "내 설정이 진짜 소설이 됐다"는 핵심 감동.
> 3. **AI 책 표지 생성** — 자랑·공유를 유발하는 시각적 완성감(바이럴 트리거).

## 4. 유저 플로우 (회원가입 → 첫 작품 1회차 + 표지 완성)

핵심 행동(Activation) = **"첫 작품의 1회차 본문 + 표지 생성 완료"**

| 단계 | 사용자 행동 | 화면 | 사용자 감정 · 기대치 |
| :---: | --- | --- | --- |
| 1 | 랜딩 진입, 예시 작품 확인 | 랜딩 | 😀 "이거 진짜 되네?" 호기심·기대 |
| 2 | 소셜 로그인(회원가입) | 가입/로그인 | 😐 "빨리 끝났으면" — 마찰 최소화 필요 |
| 3 | 온보딩 투어 + '바로 만들기' | 온보딩 | 🙂 "어렵지 않겠다" 안도 |
| 4 | 필수 설정 입력(기본:시대·인물·장르 / 서사:목표·갈등·흐름·결말 / 출력:시점·회차·등급·제목) | 설정 위저드 | 🤔 "고를 게 많네" — 4그룹·Select 추천·AI 자동채움으로 부담↓ |
| 5 | 관계도 자동 생성 확인·수정 | 관계도 빌더 | 😮 "관계가 시각화되네" 신뢰감↑ |
| 6 | '1회차 생성' 클릭, 로딩 대기 | 생성 로딩 | 😬 "잘 나올까?" 긴장·기대 (대기 UX 중요) |
| 7 | 1회차 본문 결과 확인 | 에디터/뷰어 | 🤩 **Aha! "내 설정이 소설이 됐다"** (핵심 감동) |
| 8 | 표지 생성 클릭·스타일 선택 | 표지 생성 | 😍 "표지까지?!" 완성감·소유욕 |
| 9 | 저장 → 보관함 등록 | 라이브러리 | 😌 "내 작품이 생겼다" 만족 |
| 10 | 공유 / 다음 회차 생성 유도 | 공유·결제 | 🚀 "이어서 쓰고 싶다" → 리텐션·결제 전환 |

**플로우 설계 원칙**
- 6단계(생성 대기)에서 이탈이 가장 크므로 진행률·예상시간·미리보기로 불안 해소.
- 7단계(Aha)에 도달하기까지의 입력 단계를 최소화(필수만 받고, 나머지는 기본값/추천).
- 무료 체험은 **1회차 + 표지 1회**까지 제공해 결제 전환 직전까지 가치를 체험시킴.
