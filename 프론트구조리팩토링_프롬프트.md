# 플롯위버 웹 — 프론트 구조 리팩토링 프롬프트

> **용도**: `web/` React 앱을 라우팅 기반으로 재구성하고, 레이아웃·공통 컴포넌트를 분리하며,
> 화면 프론트 설계를 최적화하고 파일 구분을 명확히 하기 위한 단일 실행 프롬프트.
> **첨부 권장**: `DESIGN.md`(토큰) + 이 문서. (개별 화면을 다시 디자인하는 게 아니라 **구조만 재편**한다.)
> **대전제**: ★회귀 금지는 구조 리팩(STEP 1–5)에만 적용. **STEP 6(디자인 통일)**은 화면 간 불일치를 수정하는 것이 목적이므로, DESIGN.md 토큰에 맞게 의도적으로 값을 통일한다.

## 0. 컨텍스트 (현재 상태 — 2026-06-29 기준, 그대로 신뢰해도 됨)

- 스택: React 18 + Vite 6 + TypeScript + Tailwind 3. 경로 `web/`.
- **라우터 미설치.** `src/App.tsx`가 `useState`와 좌하단 개발용 스위처로 **23개 화면**을 전환한다.
  - `App.tsx` 주석에 실서비스 라우트 단서 존재: `C1 /create · C2 /create/relations · C3 /create/generating`.
- 화면: `src/screens/*.tsx` **23개** (평면 구조).
- 공통 컴포넌트: `src/components/` — `WizardChrome`, `HybridSelect`, `WorkCard`, `LibraryCard`, `CoverTile`, `TipModal`, `Toast`.
- 훅: `src/lib/useViewport.ts` — `{ vw, isMobile, isDesktop }` 반환 (`isMobile = vw<768`, `isDesktop = vw>=1024`).
- 디자인 토큰: `DESIGN.md` 단일 출처 → `tailwind.config.js` / `index.css`의 커스텀 유틸(`pw-btn-*`, `brand`/`ink`/`ink2`/`wash`/`hairline` 등).

### 0-1. ★창작(C) 위저드 = 5스텝이 각각 별도 화면 (이번 리팩토링의 핵심)

| 스텝 | 화면 파일 | `WizardChrome current` | 비고 |
|---|---|---|---|
| ① 세계관 | `C0WorldWizard` | 1 | **신규.** 탭(설정/월드맵) + 우측 패널 |
| ② 기본설정 | `C1SettingsWizard` | 2 | 인물. 세계관 연동 필드(소속/지위/신분) |
| ③ 서사설정 | `CNarrativeWizard` | 3 | |
| ④ 관계도 | `C2RelationshipBuilder` | 4 | 인터랙티브 그래프 캔버스 |
| ⑤ 출력설정 | `COutputWizard` | 5 | |
| (생성) | `C3GeneratingLoader` | — | 풀스크린 로딩 |

> `WizardChrome`의 `STEPS = ["세계관","기본설정","서사설정","관계도","출력설정"]` (5개), 모바일 스텝퍼 `n / 5`.
> 다섯 스텝 화면이 **모두 `WizardChrome`(상단바+5스텝 인디케이터)를 개별 렌더**하며 `current`/`onBack`/`onSaveDraft`만 다르게 넘긴다. → **WizardLayout으로 통합할 1순위 대상.**

### 0-2. ★★C0 월드맵 ≈ C2 관계도 — 동일 인터랙션 캔버스 중복 (이번 최대 발견)

`C0WorldWizard`의 **월드맵 탭**과 `C2RelationshipBuilder`는 거의 동일한 "드래그 가능한 노드 그래프 + 우측 편집 패널" 패턴을 **각자 구현**하고 있다. 두 화면이 공유하는 로직:

| 요소 | C2 관계도 | C0 월드맵 |
|---|---|---|
| 노드 | 인물(원형 칩) | 지역(카드) |
| 노드 드래그/탭 구분 | `movedRef` + 4px 임계값 | 동일 |
| 노드 클릭→선택, **A 클릭→B 클릭=연결 생성** | ✓ (`onNodeTap`/`createOrSelectEdge`) | ✓ (`onRegionTap`/`createOrConnect`) |
| 엣지 SVG + 투명 히트패스 + 라벨칩, 클릭→선택 | ✓ | ✓ |
| 연결 대기 안내 배너 (선택 노드 강조) | ✓ | ✓ |
| 빈 곳 클릭·ESC로 선택 해제 | ✓ | ✓ |
| 우측 편집 패널(선택 대상별 분기) | EdgeEditor/NodePanel/Overview | 연결편집/지역편집/안내 |
| 캔버스 우하단 **플로팅 자동 추천** 버튼 | ✓ (`h-[40px]` 알약) | ✓ (동일) |
| 포인터 리스너(`pointermove/up`, `keydown ESC`) `useEffect` | ✓ | ✓ (복붙) |

→ **`GraphCanvas`(+ 부속) 공통 컴포넌트로 추출할 0순위 대상.** 노드/엣지 렌더러와 편집 패널만 주입형으로 분리하면 두 화면이 한 엔진을 공유한다. (STEP 4 참고)

### 0-3. 확인된 중복/안티패턴

1. 위저드 5스텝 + C3가 `WizardChrome`을 **각자 호출**(레이아웃 미부재).
2. **C0 월드맵 ↔ C2 관계도 인터랙션 캔버스 전체 중복**(0-2). 가장 큰 중복.
3. 거의 모든 화면이 `sticky top-0` **상단바/헤더를 화면 내부에 인라인 재구현**(위저드형 `h-16` / 에디터형(C4) / 표지형(C5) / 마켓형(M·D·P) `h-16` / 리더형(블러) / 랜딩형(다크 글래스) / 어드민형).
4. `Toast`/`useToast`를 **거의 전 화면이 각자 import·렌더**(전역 토스트 부재) — 최소 10개 화면 확인.
5. `useViewport`를 다수 화면이 개별 호출(레이아웃 1회로 충분한 경우 다수).
6. **크로스-화면 import**: `RadioPill`이 `C1SettingsWizard`에서 `export`되어 **`C2RelationshipBuilder`와 `COutputWizard` 두 화면이 import** 중(`import { RadioPill } from "./C1SettingsWizard"`). 화면이 다른 화면에 의존 → 공통 컴포넌트로 추출 필수.
7. `C0WorldWizard`의 **탭(설정/월드맵) 세그먼트 컨트롤**(`TabBtn`)이 인라인 — 다른 탭 화면(P2/S1 등)과 통일 가능.
8. **세계관 데이터 이중화**: C0가 세력·계급(`SAMPLES`/`KITS`/`PALETTE`)을 정의하지만, C1은 인물 연동용 세력·지위를 별도 `WORLD` 상수로 **재하드코딩**. 실제로는 C0→C1→C2로 흘러야 할 단일 출처가 화면마다 복제됨. → 공유 데이터 모듈/위저드 상태로 단일화 대상.
9. 화면 파일이 평면 배치라 **도메인(창작/마켓/서재/결제/판매자/운영) 경계가 파일 트리에 드러나지 않음**.
10. 더미 데이터 배열이 화면마다 하드코딩(C0 `KITS`/`SAMPLES`/`PALETTE`, C1 `WORLD`/`PERSONALITY_GROUPS`, CNarrative `AI_STAGES`, COutput `AI_TITLES`/`PREV`, L/M `WORKS` 등).

### STEP 2 — 레이아웃 분리
화면 내부에 인라인된 셸을 다음 레이아웃으로 추출하고, 중첩 라우트의 부모(`<Outlet/>`)로 사용한다.

- **`WizardLayout`** *(최우선)* — `WizardChrome`(상단바 + 5스텝 인디케이터)을 흡수. C0·C1·CNarrative·C2·COutput **5개 화면**이 공유하고, **`current` 단계는 라우트별로 주입**(라우트 메타 또는 `<Outlet context>`). `onBack`/`onSaveDraft` 기본 동작도 레이아웃이 제공하되 기존 토스트 문구 보존.
- `MarketLayout` — 마켓/서재/작품상세/마이/결제 공통 상단 헤더(`sticky top-0 h-16`: 로고·검색·잉크/크레딧·프로필). M·D·P 계열 헤더 단일화.
- `ReaderLayout` — 몰입형 리더 바(반투명/블러, 테마 전환). `MReader` 전용.
- `SellerLayout` — 판매자 콘솔 헤더/탭. S 계열.
- `AdminLayout` — 어드민 상단바. `OAdminConsole`.
- `PublicLayout`(선택) — 랜딩(다크 글래스 네비)/인증용 최소 래퍼.

원칙:
- 레이아웃이 `useViewport`를 **1회** 호출해 `{ vw, isMobile, isDesktop }`를 Context로 공급 → 화면 중복 호출 제거(화면은 `useViewportContext()`로 소비).
- 레이아웃은 **시각적으로 기존과 동일**해야 한다(높이·패딩·z-index·border·blur 값 보존).

### STEP 3 — 전역 토스트
- `ToastProvider` + `useToast()` 훅으로 전역화. 루트(또는 각 레이아웃)에서 1회 마운트.
- 전 화면(10+곳)의 개별 `<Toast/>` 렌더와 로컬 `useToast` import 제거, 호출부(`showToast(...)`)만 전역 훅으로 교체.
- 토스트 디자인 토큰(`toast #242537`, 모션 0.2s)·위치 그대로.

### STEP 4 — 공통 컴포넌트 승격
화면 간 반복 UI + 크로스-화면 의존을 `components/`로 추출:

- **★`GraphCanvas` (0순위, 0-2 참고)** — C0 월드맵 + C2 관계도가 공유할 인터랙티브 그래프 엔진.
  - 책임: 노드 좌표 state·드래그/탭 구분(`movedRef` 4px)·`pointermove/up`+`ESC` 리스너, 엣지 SVG(투명 히트패스+라벨칩) 클릭 선택, "노드 A→B 클릭으로 연결 생성", 연결 대기 배너, 빈 곳/ESC 해제, 우하단 플로팅 자동추천 슬롯.
  - 주입형 props: `nodes`/`edges`, `renderNode(node, {selected,dimmed})`, `nodeRadius|nodeSize`, `onConnect(a,b)`, `onSelectNode`/`onSelectEdge`, `edgeStyle`(곡선 vs 직선 점선), `autoRecommend`, 캔버스 배경(점격자 vs 양피지).
  - C2는 원형 인물 노드 + 곡선 엣지(방향 화살촉) + EdgeEditor/NodePanel, C0는 카드형 지역 노드 + 직선 점선 엣지 + 연결/지역 편집 패널을 각각 주입. **엔진은 공유, 표현만 다름.**
- **`RadioPill`** — `C1SettingsWizard`에서 `export`되어 **C2·COutput**가 import 중. → `components/ui/RadioPill.tsx`로 이동, 세 화면(C1·C2·COutput) 모두 거기서 import.
- **`SegmentedTabs`** — C0의 설정/월드맵 탭(`TabBtn`), P2·S1 등 탭 UI 통일.
- **`HybridSelect`** — 이미 분리됨(`components/`). C0·C1·CNarrative·COutput 전반에서 재사용 중 — 유지.
- `FilterPills` — `DLibrary`·`MMarketHome` 필터칩.
- `EmptyState` — 빈 서재/빈 검색/세력 0개/관계 0개 등 점선 안내 박스.
- `MobileCTABar` — `<768`에서 반복되는 하단 고정 CTA 바(위저드 5화면 동일 구조).
- `StatusBadge`/`GradeBadge` — `DWorkDetail`·`WorkCard`의 표기 요소.
- 추출 기준: **2곳 이상에서 동일/유사 마크업 반복** 또는 **크로스-화면 import**면 컴포넌트화. 1곳뿐이면 화면에 둔다(과도한 추상화 금지).
