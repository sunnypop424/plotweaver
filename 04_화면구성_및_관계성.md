# 화면 구성 및 화면 간 관계성 정립

> 두 개의 핵심 축: **① 창작(Create) 플로우** + **② 마켓플레이스(Sell/Buy) 플로우**
> 모든 UI 컴포넌트는 [컴포넌트명] — 역할 형식으로 배치 순서대로 정리

## 0. 사이트맵 (전체 화면 목록)

```
랜딩(L)
├─ 로그인/회원가입(A) → 약관 동의(A1) → [만14세미만]법정대리인 동의(A2)
├─ 성인 인증(AV) ※19금 생성·열람·판매 진입 시 트리거되는 공통 모듈
├─ [창작] 설정 위저드(C1) → 관계도 빌더(C2) → 생성 로딩(C3) → 에디터/뷰어(C4) → 표지 생성(C5)
├─ 내 작업실(D1 라이브러리) → 작품 상세/회차관리(D2)
├─ [마켓] 마켓 홈/탐색(M1) → 작품 판매페이지(M2) → 뷰어(읽기)(M3)
├─ 판매자 대시보드(S1 정산/통계) → 작품 등록·가격설정(S2)
├─ 구매내역/내 서재(B1)
├─ [신고/검수] 신고하기 모달(R1) → 내 신고내역(R2)
├─ [운영자] 검수 대시보드(O1) → 작품 검수 상세(O2) → 신고 처리(O3)
└─ 결제/요금제(P1) + 마이페이지/지갑(P2)
```

## 2-3. 성인 인증 플로우 (AV) — 19금 게이트(공통 모듈)
> **트리거 지점**: ① 19세 콘텐츠 *생성*(위저드 등급=19세 선택) ② 19금 작품 *열람/구매*(M2/M3) ③ 19금 작품 *판매 등록*(S2)
> 인증 상태는 계정에 저장 → 1회 인증 후 재요청 안 함(재인증 정책 별도).

**플로우**
```
[19금 액션 시도]
   → (이미 성인인증됨?) ──Yes──→ [정상 진행]
        │No
        ▼
   [AV 성인 인증 모달]
   1) 인증 필요 안내 + 사유
   2) 본인확인 수단 선택(휴대폰 본인인증 / 아이핀 / 신용카드 인증)
   3) 외부 인증기관(PASS 등) 호출 → 결과 수신
   → (성인 확인?) ──Yes──→ [계정 성인플래그 저장] → [원래 액션으로 복귀]
        │No(미성년/실패)
        ▼
   [차단 안내] "19세 미만은 이용할 수 없습니다" → 일반 콘텐츠로 회귀
```

| 순서 | 컴포넌트 | 역할 |
| :-: | --- | --- |
| 1 | [인증 요청 안내] | 왜 인증이 필요한지(법적 의무) 고지 |
| 2 | [본인확인 수단 Select] | 휴대폰 본인인증 / 아이핀 / 신용카드 인증 |
| 3 | [외부 인증기관 연동(PASS 등)] | 실명·연령 확인 처리 |
| 4 | [인증 결과 처리] | 성인 확인 시 계정 플래그 저장 |
| 5 | [성공 → 원액션 복귀 / 실패 → 차단 안내] | 19금 생성·열람·판매 허용 또는 차단 |
| 6 | [19금 표시 토글(마이페이지)] | 인증자라도 성인물 노출 on/off 선택 |

> **설계 원칙**: 성인 인증은 *결과 플래그만 저장*하고 인증 원본 데이터는 최소 보관(개인정보 최소수집). 미성년 우회 방지를 위해 본인 명의 인증만 허용.

## 3. 설정 위저드 (C1) — 창작 진입점

| :-: | --- | --- |
| 1 | [진행 스텝 인디케이터] | 4그룹 입력 진척 표시(이탈 방지) |
| 2 | **[그룹1 기본설정]** 배경시대·등장인물·장르 | 세계·인물·장르 골격 |

| 4 | **[그룹2 서사설정]** 목표·동기·핵심갈등·스토리흐름·결말방향 | 서사 추진력·수렴점 정의 + AI 시놉시스 추천 |
| 5 | **[그룹3]** 관계도 빌더로 이동(C2) | 인물 관계·변화 설정 |
| 6 | **[그룹4 출력설정]** 화자시점·회차·분량·연령등급·제목 | 본문 생성/판매 형식 결정 |

## 12. 창작자 통합 정산 대시보드 (S1)
> 세 수익원(판매·후원·구독연계)을 한 화면에서 관리. 각 수익에서 플랫폼 수수료 차감 후 정산.
> 진입점: 마이페이지 → '창작자 수익'. 권한: 판매/후원 등록 이력 있는 창작자.

| 순서 | 컴포넌트 | 역할 |
| :-: | --- | --- |
| 1 | [기간 필터] | 일/주/월/사용자지정 기간 선택 |
| 2 | **[총수익 요약 카드 3종]** | ① 판매 수익 ② 후원 수익 ③ 합계(정산 가능액) |
| 3 | [수익 추이 그래프] | 기간별 매출 추이(판매·후원 색 구분 스택) |
| 4 | [수익원별 비중 도넛] | 판매 vs 후원 비중 시각화 |
| 5 | **[수수료 내역 명세]** | 총 거래액 → 플랫폼 수수료(판매30%/후원10~20%) → 실 정산액 투명 표시 |
| 6 | [작품별 성과 테이블] | 작품별 조회·구매·후원수·수익 정렬 |
| 7 | [후원자 목록·메시지] | 후원자·응원 메시지 확인(팬덤 관리) |
| 8 | [정산 가능액·출금 신청] | 최소 출금액(10,000원) 충족 시 출금 |

## 20. 화면 간 관계성 (Flow Diagram)

```
[L 랜딩]
   ├─(창작)→ [A 가입] → [A1 약관동의] →([14세미만]A2 법정대리인동의)→ [C1 위저드] → [C2 관계도] → [C3 로딩]
   │            → [C4 에디터] →(표지)→ [C5 표지] → [D1 작업실]
   │                                         └─(판매등록)→ [S2 가격설정] →(사전검수 O1)→ [M2 판매페이지]
   │
   └─(읽기/구매)→ [M1 마켓홈] → [M2 판매페이지] →(미리보기)→ [결제 P1] → [M3 뷰어] → [B1 내서재]

[AV 성인인증 게이트] ※공통 모듈
   19금 액션(위저드 등급=19세 / M2·M3 19금 열람·구매 / S2 19금 판매등록)
      →(미인증 시)→ [AV 성인 인증] →(성공)→ 원래 액션 복귀 / (실패)→ 차단

[S2 등록]→[M2 노출]→[타 사용자 M3 읽기]→[리뷰/매출]→[S1 판매자 대시보드/정산]

[신고·검수 루프]
  사용자 [M2/M3] →(신고 R1)→ [검수 큐 O1] → [검수 상세 O2 / 신고처리 O3]
       → 조치(비공개·삭제·정산보류·제재) → 신고자/작성자 통보 → [R2 내 신고내역]
  자동필터·유사도검사 →(플래그)→ [O1 검수 큐]   ※ 사람 검수 트리거
```

**핵심 관계성 요약**
- **창작 플로우의 출구 = 마켓 입구**: C4(에디터) → S2(가격설정) → M2(판매)로 자연 연결되어 '생성 → 판매'가 하나의 여정.
- **마켓의 구매가 다시 창작 동기로**: 좋은 작품을 산 독자가 "나도 만들어 팔자"로 전환되는 **양면 시장(Two-sided Market) 순환 구조**.
- **공통 화면(P1 결제, P2 지갑)** 이 창작(크레딧 소모)과 마켓(구매/정산)을 모두 지탱.
- **신고·검수 루프가 마켓 신뢰를 보증**: 판매 등록(S2)은 사전검수(O1)를 거쳐 M2에 노출되고, 노출 후에도 신고(R1)→검수(O2/O3)로 상시 정화. 자동필터 플래그도 같은 큐(O1)로 합류해 **사람 검수가 최종 방어선** 역할.
