## 1. 🌟 North Star Metric (NSM)

### 1-1. 후보 비교

| 후보 | 정의 | 장점 | 약점 | 판정 |
| --- | --- | --- | --- | :---: |
| (a) **주간 회차 생성 완료 수**<br>(WCC, Weekly Chapters Completed) | 한 주간 `generation_completed`(type=chapter) 성공 건수 | 핵심 가치(=설정이 소설이 됨)와 직결, AI 원가·구독 사용량과 선형 연동, 매일 변동 관측 가능 | 순수 사용량 지표라 '완성·공유'까지의 가치는 일부만 포착 | ✅ **채택** |
| (b) 작품 완성/공유 수 | 보관함 저장 또는 공유 발생 작품 수 | 진짜 결과물·바이럴에 근접 | 후행 지표·표본 적음, 초기엔 0에 수렴해 의사결정 못 함 | 보조 |
| (c) 주간 활성 창작자 수(WAC) | 주 1회 이상 생성한 유저 수 | MAU와 직결 | 매출·원가와의 연결이 한 단계 멀다 | 보조 |

### 1-2. 추천 — **주간 회차 생성 완료 수 (WCC)**

> **NSM = 주간 회차 생성 완료 수 (Weekly Chapters Completed, type=chapter, status=done)**

**근거**
1. **가치 등식과 일치** — 플롯위버가 파는 가치는 "설정이 회차 본문이 되는 경험"(`01` Aha). 이 행동의 빈도가 곧 전달된 가치량이다.
2. **3대 사업 변수와 동시 연동** — 생성량↑은 (MAU↑ × 활성도↑)의 결과이자, 크레딧 소비→구독 전환 압력의 선행 신호. `08`의 *MAU>전환율>단가* 체인을 한 숫자로 압축한다.
3. **원가 리스크의 분모** — 생성 1건마다 원가 발생(`12` gen_cost). NSM이 곧 원가 발생 횟수이므로 **NSM ÷ 원가**를 항상 같이 봐야 한다(헛성장 방지 → 5장 가드레일과 짝).
4. **조작 저항** — '완료(성공)' 건만 카운트해 실패·어뷰징 허수를 배제.

**NSM 분해식(움직이는 레버 확인용)**
```
WCC = 활성 창작자 수(WAC)
      × 창작자당 주간 생성 시도 수
      × 생성 성공률(=1 − 생성 실패율)
```
→ NSM이 흔들리면 위 3개 레버 중 어디서 깨졌는지 즉시 분해한다.

### 1-3. 보조 지표 세트 (NSM 健全성 검증)

| 구분 | 지표 | 역할 |
| --- | --- | --- |
| 폭(Breadth) | 주간 활성 창작자 수(WAC) | 소수 헤비유저 의존 여부 점검 |
| 깊이(Depth) | 창작자당 주간 회차 수 | 인당 가치 사용량 |
| 품질(Quality) | 생성 성공률 / 회차 재생성율 | "많이 생성=좋음"의 함정 차단 |
| 가치확정 | Activation 완료율 / 작품 저장·공유율 | 생성이 '완성'으로 연결되는지 |
| 경제성 | 생성당 원가·구독 전환율 | NSM이 매출·이익으로 환산되는지 |

## 2. 🔻 AARRR 퍼널 × 유저플로우 10단계 매핑

`01` 유저플로우 10단계를 AARRR에 매핑하고, 각 단계의 핵심 지표·이탈 원인을 정의한다.

| 플로우 단계 | AARRR | 핵심 이벤트 | 핵심 지표 | 주 이탈 원인·대응 |
| :---: | --- | --- | --- | --- |
| 1 랜딩·예시 확인 | **Acquisition** | `landing_view` | 방문수, 예시 열람율, CPL | 가치제안 불명확 → 예시 결과물·표지 노출 |
| 2 소셜 로그인 | Acquisition→Activation | `signup_started`·`signup` | **가입 전환율**, 소셜 성공률 | 가입 마찰 → 소셜 3종, 단계 최소화(`F-01`) |
| 3 온보딩 투어 | Activation | `onboarding_complete` | 온보딩 완료율, '바로 만들기' 클릭률 | 투어 길면 이탈 → 스킵 허용 |
| 4 설정 위저드 | Activation | `wizard_step_complete` | **스텝별 통과율**, 위저드 완주율 | "고를 게 많다"(`01`) → Select 추천·AI 자동채움, 필수만 |
| 5 관계도 확인·수정 | Activation | `relationship_built` | 관계도 통과율 | 복잡도 부담 → 자동 생성값 그대로 진행 허용 |
| 6 **1회차 생성 클릭·대기** | **Activation (최대 이탈)** | `generation_started` | **생성 시작→완료 도달률**, 대기 이탈율, 생성 지연(p50/p95) | ⚠️ **이탈 1순위**: 로딩 불안 → 진행률·예상시간·미리보기, 실패 시 자동 재시도 |
| 7 **1회차 결과 확인 (Aha)** | **Activation (Aha)** | `generation_completed` ★ | **Aha 도달률 = 생성완료자/가입자**, 결과 체류시간 | 결과 품질 실망 → 품질 가드레일·재생성 유도 |
| 8 표지 생성·선택 | Activation | `cover_generated` | 표지 생성율(Aha 도달자 중) | 스타일 불만 → 후보 N개·재생성(`F-14`) |
| 9 저장→보관함 | **Activation 완료** | `work_saved` | **Activation 완료율**(=본문+표지+저장) | — (여기까지가 Activation 정의 충족) |
| 10 공유·다음 회차·결제 유도 | **Retention/Revenue/Referral** | `share_clicked`·`paywall_viewed`·`subscribe` | 공유율, 페이월 노출→전환율, 차회차 생성율 | 결제 저항 → 무료 후원 우회(`05`), 가치 체험 후 페이월 |

### 2-1. 이탈 집중 관리 2개 게이트

> **게이트 1 — 6단계 생성대기 이탈**
> 측정: `1 − (generation_completed / generation_started)`. 생성 지연(p95)과 강한 상관. 목표: 대기 이탈율 ≤ 15%. 위젯: 제품 대시보드 상시 노출.

> **게이트 2 — 7단계 Aha 도달률**
> 측정: `generation_completed 유저(distinct) / signup 유저(distinct)` (가입 코호트 기준). 이 한 숫자가 Activation·리텐션·매출의 최상류 선행 지표. 목표: 가입자 대비 Aha 도달 ≥ 55%.

### 4-4. 원가 대시보드 (Cost) — ⚠️ 1순위 리스크 전담
> `08`·`09`의 최치명 리스크("AI 원가 > 판매가")를 상시 감시. 데이터 원천 = `chapters.gen_cost/gen_model`, `generation_jobs`.

| 위젯 | 정의 |
| --- | --- |
| **생성당 평균 원가** | `Σ gen_cost / 생성완료 건수` (type별: chapter/cover) |
| **원가율 (생성원가 ÷ 크레딧 환산매출)** | 100% 초과 = 구조적 적자 경보 |
| **생성 성공률 / 실패율** | `done / (done+failed)`, 실패는 원가 낭비+크레딧 롤백 |
| **생성 지연 p50·p95·평균** | `latency_ms`, 6단계 이탈과 연동 |
| 모델별 원가·품질 비교 | `gen_model`별 단가 vs 재생성율 (다운그레이드 의사결정) |
| 유저당 월 생성원가 vs ARPU | 헤비유저 적자 구간 탐지 |
| 무료유저 누적 생성원가 | 무료 남용/어뷰징 비용 감시 |

## 5. 🚧 가드레일 지표 (성장의 부작용 차단)

> 성장(NSM·전환) 추구가 **품질·이익·신뢰**를 깨뜨리지 않는지 감시. 임계 초과 시 실험 중단·롤백.

| 영역 | 지표 | 측정식 | 경보 임계(예시) |
| --- | --- | --- | --- |
| 품질 붕괴 | 회차 재생성율 | 재생성 요청 / 생성완료 | > 30% |
| 품질 붕괴 | 30화 일관성 붕괴율 | 설정붕괴·반복 플래그 / 검수 표본 | > 20% (`09` H2 기준) |
| 생성 안정성 | **생성 실패율** | `generation_failed / generation_started` | > 5% |
| 생성 안정성 | 생성 지연 p95 | `latency_ms` p95 | > 60초 |
| 경제성 | **원가율** | 생성원가 / 크레딧 환산매출 | ≥ 100% (즉시 차단) |
| 경제성 | 무료유저 1인당 원가 | 무료 생성원가 합 / 무료 MAU | 예산 대비 초과 |
| CS | CS 인입율 | 문의 티켓 / 활성유저 | 급증(주간 2배) |
| 신뢰·안전 | **신고율** | `content_reported` / 공개 작품 | > 1% |
| 신뢰·안전 | 모더레이션 차단율 | `F-40` 플래그 / 생성건 | 급증 |
| 결제 건전성 | 환불·자전거래 의심율 | FDS 홀드 / 거래 | > 임계(`F-43`) |

## 6. 🧮 핵심 비율 지표 정의식

> 모든 비율은 **분자/분모/기준 코호트**를 못박는다. 모호하면 측정 불가.

| 지표 | 정의식 | 기준 |
| --- | --- | --- |
| **가입→Activation 전환율** | `Activation 완료 유저(본문+표지+저장) / 가입 유저` | 가입일 코호트, 7일 내 |
| Aha 도달률(게이트2) | `generation_completed(distinct) / signup(distinct)` | 가입 코호트 |
| 6단계 대기 이탈율 | `1 − generation_completed / generation_started` | 일 단위 |
| **무료→유료 전환율** | `subscribe 유저 / 무료 활성 유저` | 월 코호트, 목표 ≥ 3%(`09`) |
| **회차 재생성율** | `regeneration_requested / generation_completed(chapter)` | 주간 |
| 생성 성공률 | `done / (done + failed)` | 주간 |
| **D1 / D7 / D30 리텐션** | `가입 후 N일째 재방문(또는 생성·열람) 유저 / 해당일 코호트 가입자` | 가입일 코호트 |
| **ARPU** | `월 매출(구독+판매수수료+후원수수료) / MAU` | 월 |
| **ARPPU** | `월 구독매출 / 유료 구독자 수` | 월, `08`은 12~16천원 가정 |
| LTV(근사) | `ARPPU × 평균 구독 유지개월(1/월해지율)` | 월 |
| 공유율(Referral) | `share_clicked 유저 / Activation 완료 유저` | 주간 |
| 후원 전환율 | `tip_sent 독자 / 작품 열람 독자` | 목표 ≥ 1%(`09` H4) |

> 매출 환산은 항상 `08` 등식과 정합: **구독매출 = MAU × 구독전환율 × ARPPU**. 대시보드 숫자와 시뮬레이션 가정을 같은 식으로 맞춰 검증/보정한다.

## 7. 🧪 베타 실험(`09`) 가설 ↔ 측정지표 연결

| # | 가설 | 측정 지표(본 문서) | 이벤트 원천 | 성공 기준(`09`) |
| :-: | --- | --- | --- | --- |
| **H1** | 사람들이 AI 소설 생성을 원한다 | 랜딩→가입 전환율, CPL | `landing_view`→`signup` | 전환율 ≥ 8%, CPL ≤ 3,000원 |

- **원가 영향 필수 검토** — 생성·모델 관련 실험은 원가율 변화를 항상 함께 리포트.

### 8-2. 실험 우선순위 (ICE 약식: Impact·Confidence·Ease)

## 9. 🛠️ 도구 · 구현

### 9-1. 이벤트 수집 스택(제안)
- **제품 분석**: 제품 이벤트 트래커(Amplitude/PostHog 등 1종) — 퍼널·리텐션·코호트. 초기 비용·셀프호스팅 고려 시 **PostHog** 권장(이벤트+세션+실험 통합).
- **서버 이벤트(신뢰원천)**: `generation_started/completed/failed`, `subscribe`, `*_purchase`, `tip_sent`는 **백엔드(워커·결제 콜백)에서 발화** → 클라 유실/조작 방지.
- **데이터 웨어हाउस**: 운영 DB(`12`) → 일배치 적재(BigQuery/Postgres 분석 복제본) → 대시보드(Metabase/Looker). 가드레일·원가는 **준실시간**(쿼리 스케줄 15~60분).
- **태깅 거버넌스**: 본 문서의 택소노미(3장)를 단일 스펙으로, 신규 이벤트는 PR 리뷰로 등록.

### 9-2. 원가·품질 로깅 활용 (`12` generation_jobs / chapters)
- 원천: `chapters.gen_cost`·`gen_model`, `generation_jobs.status`·`progress`·`error`. (`12`: "gen_cost/gen_model 로깅으로 원가 모니터링".)
- **파생 지표 파이프라인**:
  - 생성당 원가 = `AVG(gen_cost) GROUP BY type`
  - 원가율 = `SUM(gen_cost) / 크레딧 환산매출`
  - 실패율 = `count(status='failed') / count(started)` (실패는 크레딧 롤백이라 매출 0 + 원가 낭비 → 별도 추적)
  - 지연 p95 = `generation_completed.latency_ms` 분위수
- **모델/단가 config 연동**: 본문·표지 모델·단가·분량이 코드가 아닌 config(`09`·`11`)이므로, `gen_model` 속성으로 **config 변경 전후 A/B 원가·품질 비교**가 자연히 성립.
