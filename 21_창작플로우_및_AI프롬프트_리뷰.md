## 1. 전체 플로우 지도

```
/create/world      (C0 세계관)      saveWorld      →
/create            (C1 인물)        saveCharacters →
/create/relations  (C2 관계도)      saveRelations  →
/create/narrative  (C4 서사)        saveNarrative  →
/create/output     (C5 출력·제목)   saveOutput     → createNovel() → /create/generating
                                                                        ↓
                                                              C3 생성 로더 (회차 생성)
                                                                        ↓
                                                        /works/:id/edit (C4 에디터)
                                                          ├─ /works/:id/cover (C5 표지)
                                                          └─ /seller/register (판매등록)
```

- 라우트 경로와 단계 번호가 어긋난다 — 2단계(인물)가 `/create`라는 "루트" 경로를 차지하고, 1단계(세계관)가 오히려 `/create/world`라는 하위 경로다(기능상 문제는 없음, 현행 유지).
- 위저드는 하나의 거대한 `WizardState`(= `NovelSettings` + 세션/드래프트용 부가 필드)를 `WizardProvider`가 들고 있고, 각 페이지는 자기 구간만 `merge()`로 갱신한다.
- **[2차 개정 완료] "설정 편집"으로 위저드에 재진입한 뒤, 최종 단계(COutput)까지 가지 않고 중간에 이탈하면 변경분이 DB에 전혀 반영되지 않던 문제** — C0~CNarrative의 "다음" 버튼(총 8곳)이 편집모드일 때 단계마다 즉시 `updateNovel`을 호출하도록 수정. 실제 브라우저로 세계관 문구를 바꾸고 "다음"만 누른 뒤(최종 저장 전) DB를 직접 조회해 이미 반영돼 있음을 확인했다.

## 3. 백엔드 프롬프트 아키텍처 (회차 생성 · 표지 생성)

`engine_poc/prompts.py`가 회차 생성 프롬프트를 3계층으로 구성한다.

1. **Layer 1 — `build_system()`**: "작품 헌법" + 인물카드(외형·특징·성격·욕망·공포·말버릇·비밀) + 파워시스템/감정목표/회차패턴 블록 + 세계관 규칙(세력·계급·의상·지역·지역연결·세력관계·용어사전이 하나의 문자열로 합쳐진 것) + "집필 원칙" 7개 조항(클리프행어 스타일은 알려진 4종 밖의 값이어도 범용 폴백 규칙이 붙는다).
2. **Layer 2 — `build_context()`**: 기승전결 골격, 관계 타임라인 해석 텍스트, 직전 최근 3회차 요약, 열린 복선(회차마다 누적·병합, 최대 15개 캡), 예정된 복선, 고유명사 표기(위저드 용어사전으로 최초 1회 시딩 + 회차마다 자동 추출된 신규 용어가 병합됨).
3. **Layer 3 — `build_task()`**: 이번 회차 번호/단계, 직전 화 마지막 장면, 작가의 자유 지시(`user_note`), "이제 N화를 써라".

**표지 생성**(`generate_cover`)은 별도 파이프라인(`gpt-image-2`)으로, 인물 외형·의상(세력 costume)·관계 무드(`REL_MOOD`)에 더해 이번 2차 개정으로 **세력관계 무드, 감정목표, 레퍼런스 분위기, 클리프행어 구도, 파워시스템 등급, 거점 지역 분위기**까지 조건부로 반영하도록 확장됐다.

**이 구조 자체는 이 서비스의 핵심 차별화 포인트다.** 관계 타임라인 → "지금부터 복선을 깔 것" 지시문 자동 변환이 여전히 가장 정교한 부분이며, 두 차례의 개정을 거치며 위저드에서 입력하는 사실상 모든 데이터(월드맵, 세력관계, 심리 프로필, 복선계획표, 작가 지시, 감정목표·레퍼런스·클리프행어)가 실제로 프롬프트(회차 본문 또는 표지)에 도달하게 됐다.

## 5. 2차 개정 — 연결성 매트릭스 전강화 + 잔여 이슈 처리 (완료·검증됨)

1차 개정 이후에도 매트릭스에 남아있던 약한/없는 칸(◐/○)을 전부 강함(●)으로 올리고, 1차 개정 문서가 범위 밖으로 남겼던 3가지를 마저 처리했다. 모두 **실제 서버 재기동(reload-dir 포함) + 실제 Anthropic/OpenAI API 호출(회차 생성 2회, 표지 생성 1회, suggest 호출 다수) + Supabase DB 직접 조회 + 실제 브라우저(Playwright)** 로 검증했다.

| # | 문제였던 것 | 수정 내용 | 검증 결과 |
|---|---|---|---|
| A0 | `C5CoverGenerator`가 `coverStyle`은 복원하는데 `coverTone`은 항상 `"어두운"`으로 시작 → 표지 재생성 시 저장된 톤을 **조용히 덮어쓰는 데이터 유실 버그** | 마운트 시 `coverTone` 복원 블록 추가 (`style`과 동일 패턴) | `coverTone: "밝은"`으로 저장된 작품을 브라우저에서 열자 톤 드롭다운이 실제로 "밝은"으로 표시됨을 확인(덮어쓰기 오판 조건 해소) |
| A1 | `suggest_relations`가 세계관(세력관계 등)을 전혀 안 받음 | `worldRules`/`factionRelations`(세력 id→이름 변환 후 전달) 추가, 캐릭터에 `faction` 필드 추가 | 실제 호출 시 백엔드 로그에서 `world_context`에 "화산파 — 흑풍채: 적대" 등 세력관계 문구가 정확히 조립됨을 확인 |
| A2 | `suggest_title`이 세력·관계·감정목표·레퍼런스·클리프행어를 안 씀 | 5개 필드 추가, 조건부 프롬프트 라인 삽입 | 실제 호출 시 백엔드 로그에서 5개 라인이 모두 정확한 값으로 삽입됨을 확인 |
| A3 | 표지 생성이 세력관계·감정목표·레퍼런스·클리프행어·파워시스템·거점지역을 안 씀 | `generate_cover`의 `narrative_hint_parts`에 6개 힌트를 조건부로 추가(세력관계 무드 dict, 감정목표 dict, 레퍼런스 분위기, 클리프행어 구도 dict, 파워티어, 거점지역 묘사) | 실제 이미지 생성 1회 호출 → 최종 프롬프트 문자열에 6개 힌트 전부(예: `"a hostile, warlike tension between rival factions"`, `"triumphant, exhilarating energy"`, `"visual cues suggesting the protagonist's power tier: 일류"` 등)가 정확히 포함되고 실제 이미지가 생성됨을 확인 |
| B | 스토리 바이블 `summaries` 무한 누적 | `writeback()`에서 최신 회차를 제외한 나머지의 `last_scene`(회차당 최대 280자)을 비움 — `assemble()`이 가장 최근 것만 읽는다는 점 재확인 후 적용 | 회차 1·2 생성 후 DB 조회: seq=1의 `last_scene` 길이 0, seq=2(최신)만 280자 유지됨을 확인 |
| C | `uvicorn --reload`가 `engine_poc/`를 안 감시 | `RUN.md` 백엔드 실행 커맨드에 `--reload-dir . --reload-dir ../engine_poc` 추가 | 새 커맨드로 기동 후 `engine_poc/engine.py`를 실제로 수정 → 수동 재시작 없이 `WatchFiles detected changes ... Reloading` 로그로 자동 반영 확인 |
| D | 편집기→위저드 재진입 중 최종 단계 전 이탈 시 변경분이 DB에 전혀 반영 안 됨 | C0~CNarrative "다음" 버튼(8곳)이 편집모드일 때 `updateNovel`을 즉시 호출(실패 시 토스트+이동 차단) | 브라우저에서 세계관 문구에 마커 텍스트를 추가하고 "다음"만 클릭(최종 저장 전) → Supabase REST로 직접 조회해 마커가 이미 DB에 반영돼 있음을 확인 |

**검증 중 발견한 부수 사항**: 첫 검증 시도에서 `uvicorn --reload`(reload-dir 미적용 상태)가 `engine_poc/` 변경을 못 잡는 것을 실제로 재현했고, 이는 항목 C가 고치는 바로 그 문제였다 — 수정 전/후 동작 차이를 직접 비교한 셈이라 항목 C의 근거가 한 번 더 실증됐다.

## 6. 위저드 단계 간 유기적 연결성 매트릭스 (2차 개정 — 전면 강화)

`●` 강함(구조화된 데이터가 그대로/의미있게 전달) · `◐` 약함 · `○` 없음. 이번 2차 개정으로 새로 강해진 셀은 **굵게** 표시.

## 7. 결론

두 차례의 개정을 거쳐 리뷰가 지적한 위저드→프롬프트 연결성 문제를 사실상 전부 해소했다: 1차에서 13개 개별 항목을, 2차에서 매트릭스에 남아있던 모든 약한/없는 연결과 잔여 3항목(스토리 바이블 누적, 편집모드 실시간 동기화, reload-dir)을 처리했다. 매 항목을 코드 리뷰가 아니라 **실제 서버·실제 AI 호출(Anthropic 텍스트 생성 + OpenAI 이미지 생성)·실제 브라우저·DB 직접 조회**로 재현해 확인했다 — 계획한 복선이 지정 회차에 정확히 회수되고, 작가 자유 지시가 생성 제목에 그대로 반영되고, 편집모드에서 최종 저장 전에 이미 DB가 갱신되는 것까지 전부 실측했다.

남은 것은 두 가지뿐이며 둘 다 경미하다: ① C0 단계의 "목표 정보 없음" 폴백은 위저드 순서상 구조적 한계로 이 이상 개선 여지가 없고, ② 스토리 바이블 `summaries` 목록 자체의 장기 압축은 사용자 체감 영향이 없는 순수 DB 최적화라 후순위로 미뤄도 무방하다.
