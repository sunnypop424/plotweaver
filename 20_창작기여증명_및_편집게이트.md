# 창작 기여 증명 & 편집 게이트 (Authorship & Creative Contribution System)

> "설정 입력만으로는 저작권이 자동 인정되지 않을 수 있다" 문제의 정면 해결책
> 작성: 수석 PM / 최종수정: 2026-06-29

---

## 0. 🎯 이 문서가 푸는 문제

`19_AI생성물_저작권_및_콘텐츠정책` §1-2·§2-2의 핵심 함의:

> ⚠️ **"설정 입력만"으로는 저작권이 자동 인정되지 않을 수 있다. 보호 가능성을 높이려면 사용자의 선택·편집·증감(에디터 작업)이라는 추가적 창작 기여가 필요하다.**

그런데 현재 정책은 **"마켓 판매 전 1회 이상 에디터 편집을 *권장*"**(19 §2-2) 수준이다. 권장은 다음 이유로 불충분하다.

| 약점 | 결과 |
| --- | --- |
| **권장은 강제력이 없다** | 사용자가 설정만 넣고 AI 본문 그대로 판매 → 저작물성 약한 작품이 마켓에 유입 → 수익모델의 법적 전제 붕괴 |
| **기여를 기록하지 않는다** | 분쟁·저작권 등록 시 "이 사람이 실제로 창작 기여를 했다"는 **증거가 없다** |
| **기여를 측정하지 않는다** | "AI 생성"인지 "AI 보조(사용자 편집)"인지 라벨(19 §5)을 객관적으로 정할 수 없다 |

> 💡 **해결 전략 한 줄**: 창작 기여를 **(1) 자연스럽게 유도하고 → (2) 자동으로 측정하고 → (3) 판매 게이트로 강제하고 → (4) 증거로 보존**하는 시스템을 만든다. "권장"을 **"측정 가능한 게이트 + 증거 로그"**로 승격한다.

> ⚖️ 본 문서는 법률자문이 아니다(19 면책 고지 준용). 이 시스템은 **저작권 인정을 보장하지 않으며**, 인정 **가능성을 높이고** 분쟁 시 **방어 증거를 확보**하는 리스크 저감 장치다. 【확인 필요: 임계치·라벨 기준의 법적 적정성은 법무 검토】

## 1. 🧱 시스템 4대 구성요소

```
 ┌──────────────────────────────────────────────────────────────┐
 │  ① 유도(Encourage)   에디터가 편집을 '쉽고 의미있게' 만든다       │
 │        │                                                       │
 │        ▼                                                       │
 │  ② 측정(Measure)     모든 편집·선택·증감을 자동 캡처 → 기여도 산출 │
```

## 2. ① 유도 — 편집을 쉽고 의미있게 (Editor, `05` C4 확장)

기여를 강제하기 전에, **편집이 부담이 아니라 가치가 되도록** 에디터(C4)를 설계한다. "마지못해 한 글자 고치기"가 아니라 **실제 창작 행위**가 일어나게 유도한다.

| 에디터 기능 | 창작 기여 성격(19 §2-2) | 측정 연동 |
| --- | --- | --- |
| **문단 직접 작성 블록**("내 문장 추가") | 표현의 증감 — *가장 강한 기여* | `human_block` 카운트 |
| **인라인 재작성·톤 조정**(선택 구간 다시 쓰기) | 표현의 수정 | `edit_ratio` 반영 |
| **회차/문단 후보 중 선택**(N개 재생성 비교) | 선택·구성(편집저작물적) | `candidate_select` |
| **구조 편집**(관계 타임라인·비트·인물카드 수정) | 선택·배열 | `structural_edit` |
| **삭제·재배열**(문단 순서·분량 조정) | 구성 | `edit_ratio`·`structural_edit` |

**유도 UX 원칙**
- 편집 시 **실시간 기여도 게이지**를 보여준다("판매 가능 기준까지 32%"). 게임화로 거부감↓.
- "AI 초안 → 내 작품으로" 전환을 **명시적 단계**로 제시(초안 검토 → 편집 → 완성).
- 자유·비공개 보관은 **편집 강제 없음**(마찰 0). 게이트는 **유료 판매 시에만** 작동(§4).

## 3. ② 측정 — 기여도 자동 산출 (Contribution Scoring)

### 3-1. 무엇을 측정하나 (캡처 대상)

에디터의 모든 행위를 **append-only 로그(`authorship_events`, §5)**로 캡처하고, 회차별 **기여 지표**를 계산한다.

| 지표 | 정의 | 계산 방법 |
| --- | --- | --- |
| `edit_ratio` | AI 초안 대비 사람이 바꾸거나 더한 비율 | 초안(v0) vs 최종(vN) **토큰/문자 diff** → (변경+추가)/최종 길이 |
| `human_blocks` | 사람이 처음부터 작성한 문단 수 | "내 문장 추가" 블록 카운트 |
| `revision_count` | 실질 변경이 있는 저장 횟수 | 사소 변경(오탈자·공백) 제외(19 §1-2 불인정 예시 반영) |
| `candidate_selections` | 복수 후보 중 선택한 횟수 | 재생성 후보 비교·채택 이벤트 |
| `structural_edits` | 관계도·비트·인물카드 등 구조 편집 수 | 구조 편집 이벤트 |

> ⚠️ **사소 변경 배제**가 핵심이다. 19 §1-2는 "오탈자 수정·단순 색상 변경"을 창작 기여로 **불인정**한다. 따라서 `edit_ratio`는 의미있는 변경만 카운트하고, 공백/문장부호/한두 글자 치환은 가중치 0 처리한다.

### 3-2. 기여도 점수 & 등급 (Tier)

회차별 가중합으로 `contribution_score`(0~100)를 내고 **3등급**으로 분류한다.

```
score = w1·norm(edit_ratio) + w2·norm(human_blocks) + w3·norm(structural_edits)
        + w4·norm(candidate_selections) + w5·norm(revision_count)
   (가중치 w·정규화 함수는 PoC로 튜닝 — 【확인 필요】)
```

| 등급(tier) | 의미 | 라벨(19 §5) | 유료 판매 |
| --- | --- | --- | :---: |
| `ai_generated` | 사람 기여 미미(설정+그대로) | **"AI 생성"** | ❌ 차단 |
| `ai_assisted` | 의미있는 편집·선택 존재 | **"AI 보조 (사용자 편집)"** | ✅ 가능 |
| `user_led` | 편집·증감 비중 높음 | **"사용자 주도 (AI 보조)"** | ✅ 가능 |

> 등급이 곧 **판매 게이트(§4)** 와 **고지 라벨(19 §5)** 을 동시에 결정한다. 즉 "기여를 한 만큼 권리가 강해지고, 그만큼 라벨이 바뀌고, 판매가 열린다"는 **일관된 인센티브 구조**.

### 3-3. 본문 vs 표지 분리 (19 §2-3 정합)

| 대상 | 측정 방식 | 비고 |
| --- | --- | --- |
| **본문(텍스트)** | §3-1 지표 전체 적용 | 편집 기여 측정이 비교적 명확 |
| **표지(이미지)** | 후보 선택·요소 편집(타이포 후합성·크롭·합성)만 기여로 인정 | 19 §2-3대로 **독점 보호 약함**을 별도 고지. 표지는 등급 산정에서 **본문과 분리** |

## 4. ③ 게이트 — 기여도 미달 시 유료 판매 차단

> 【정책 결정】 **유료 판매(`for_sale`) 등록 시**, 판매 대상 회차는 **`ai_assisted` 이상**이어야 한다. `ai_generated` 회차가 하나라도 포함되면 등록을 **차단**한다. (자유·무료·비공개는 게이트 없음)

이로써 19 §2-2의 "권장"이 **검증 가능한 강제 게이트**로 승격된다.

### 4-1. 판매 등록 플로우 (19 §3-2 게이트 보강)

```
[유료 판매 등록 요청]  (POST /novels/{id}/publish, visibility=paid)
   ▼
[기여도 게이트]  판매 회차들의 tier 검사
   ├─ 미달(ai_generated 포함) → 403 CONTRIBUTION_REQUIRED
   │        ▼
   │   에디터로 유도: "판매하려면 직접 편집이 필요합니다"
   │   + 부족 회차·부족분(%) 표시 + 빠른 편집 진입
   │
   └─ 통과(전 회차 ai_assisted 이상)
            ▼
      [기존 판매 게이트] 등급·성인인증·F-50 유사도·라벨 동의(19 §3-2)
            ▼
      [검수 큐 → 마켓 노출]
```

### 4-2. 게이트 정책 세부

| 항목 | 정책 |
| --- | --- |
| 적용 범위 | **유료 판매 등록 시점**. 무료 공개·비공개·보관은 미적용 |
| 검사 단위 | 판매에 포함되는 **회차별** 검사(묶음 판매면 전 회차) |
| 임계치 | `ai_generated`→`ai_assisted` 경계값. **초기 보수적 설정 후 PoC·운영 데이터로 튜닝** 【확인 필요】 |
| 표지 | 표지 단독은 판매 대상 아님. 작품에 종속되며 라벨·고지로 처리(19 §2-3) |
| 우회 방지 | 기여도는 **서버에서 재계산**(클라 신뢰 금지). diff·이벤트 로그 기반 |
| 예외 | 사용자가 **처음부터 직접 집필**(AI 미사용) → 당연히 통과(human_blocks 우위) |

> ⚠️ 【리스크】 게이트가 **전환의 마지막 단계(판매)에 마찰**을 준다. 완화책: 게이트를 **에디터 단계에서 미리 게이지로 노출**(§2)해 판매 시점에 놀라지 않게 한다. 또한 임계치를 **너무 높이면 이탈**, **너무 낮으면 보호 효과 약화** — A/B로 균형점 탐색.

## 5. ④ 증거 — 편집 이력·기여 내역서 보존 (Provenance)

저작권 등록·분쟁에서 결정적인 것은 **"인간의 창작적 기여가 실제 있었다"는 입증**이다. 이를 위해 **불변(append-only) 이력**을 남긴다.

### 5-1. 보존 데이터 (DB는 `12` 반영)

| 데이터 | 내용 | 용도 |
| --- | --- | --- |
| `chapter_revisions` | v0(AI 초안)~vN(최종)의 스냅샷/diff | 편집 전후 비교 = 기여 증거 |
| `authorship_events` | 생성·편집구간·후보선택·문단추가·구조편집 이벤트(append-only) | **타임스탬프 있는 창작 행위 기록** |
| `contribution_scores` | 회차/작품별 기여 지표·점수·등급 | 라벨·게이트 근거, 내역서 산출 |

### 5-2. 창작 기여 내역서 (Authorship Statement) — 산출물

작품/회차 단위로 **사람이 읽을 수 있는 기여 내역서**를 생성·내보내기(PDF/JSON)한다.

```
── 창작 기여 내역서 (작품: "회귀한 검사") ──────────────
 작성자: user_xxx        생성 도구: 플롯위버(AI 보조)
 회차    AI초안→최종 변경률  사람작성 문단  구조편집  등급
  1화        34%             3            2       AI보조
  2화        51%             5            1       사용자주도
 ...
 첨부: 회차별 편집 전/후 스냅샷, 편집 타임라인(이벤트 로그)
 ※ 본 내역서는 인간의 편집·선택·증감 기여를 기록한 자료입니다.
   (저작권 인정 여부는 별도 판단 — 19 §2-4)
─────────────────────────────────────────────
```

용도:
- **저작권 등록** 신청 시 "창작적 기여" 소명 자료(국내 안내서 취지 — 19 §1-2).
- **표절·침해 분쟁** 시 작성자 방어 증거(19 §6-3·§7-2 Notice & Takedown 소명).
- 저작물성이 부정되더라도 **부정경쟁방지법·계약 위반** 등 다른 법리의 사실관계 입증에 활용(19 §2-4 【리스크】 보완).

### 5-3. 무결성 원칙

- `authorship_events`는 **append-only**(수정·삭제 불가). 위·변조 방지.
- 기여도는 **이벤트 로그로부터 서버가 재계산**(저장값 신뢰 X).
- 보존 기간: 작품 게시·판매 지속 기간 + 분쟁 대비 기간(개인정보방침 `07` E 정합 — 【확인 필요: 보존기간】).

## 7. 🧪 PoC·운영에서 정할 것 (오픈 이슈)

- [ ] 기여 지표 **가중치·정규화·임계치**(ai_generated↔ai_assisted 경계) — 데이터로 튜닝.
- [ ] 게이트가 **전환율에 주는 영향** A/B(임계치 높낮이 vs 이탈·보호효과).
- [ ] "사소 변경 배제" 판정 로직(공백·오탈자·N글자 미만)의 정밀도.
- [ ] 표지 기여 인정 범위(후합성·크롭·합성 어디까지).
- [ ] 내역서 양식의 **법적 활용 적정성**(등록·분쟁) — 법무 확인.
- [ ] 이력 보존 기간·개인정보 정합(`07` E).
- [ ] 라벨 3등급 명칭의 소비자 오인 가능성(표시광고·기만 — 19 §5).

## 8. ✅ 요약 — "권장"에서 "보장 가능한 시스템"으로

| Before (19 §2-2) | After (본 문서 F-16) |
| --- | --- |
| 편집을 **권장**(강제력·기록 없음) | 편집을 **유도→측정→게이트→증거**의 시스템으로 |
| 기여 여부 **불명** | 기여도 **정량 점수·3등급** |
| 판매 시 **무방비로 AI 생성물 유입** | `ai_assisted` 미만 **유료 판매 차단** |
| 분쟁 시 **증거 없음** | **불변 편집 이력 + 기여 내역서** |
| 라벨 **주관적** | 기여 등급으로 **라벨 자동 결정** |

> 이로써 "설정 입력만으로는 저작권이 약하다"는 리스크를, **사용자가 실제 창작 기여를 하도록 시스템이 유도·강제하고 그 증거를 남기는 구조**로 정면 대응한다. (단, 저작권 인정 자체를 **보장하지는 않으며** 가능성을 높이는 장치임 — 19 §2-4)
