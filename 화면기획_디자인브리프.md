# 플롯위버 — 화면 기획 & 디자인 브리프 (Front Design Input)

> **용도**: Claude 디자인 입력용 **단일 문서**. 비주얼 토큰(색·타이포·간격)은 별도 `design.md`를 따르고, 본 문서는 **무엇을·어떻게 배치/동작**하는지를 정의한다.
> **통합 출처**: `04`(화면구성)·`10`(와이어프레임)·`02`/`03`(입력 항목)·`11`(기능 동작)·`05`(수익).
> **범례**: `[버튼]` `▼Select` `____입력` `(•)라디오` `[x]체크` `🖼이미지` `≡목록` / 우선순위 **P0**(MVP)·**P1**(마켓)·P2(이후)

---

## 0. 제품 한 줄 & 디자인 원칙

**"설정만 넣으면 AI가 회차별 웹소설과 책 표지까지 완성해 주고, 만든 작품으로 후원·판매까지 하는 창작 스튜디오."**

| # | 디자인 원칙 | 적용 |
| :-: | --- | --- |
| 1 | **막막함 제로** | 위저드는 Select 추천값 우선 + 빈칸엔 AI 자동채움. 백지 공포 제거 |
| 2 | **Aha를 빨리** | 가입→1회차 본문+표지까지 최단 동선. 입력 피로 최소화 |
| 3 | **대기 불안 해소** | 생성(20~40초)은 진행률·예상시간·설정요약·팁으로 채움 |
| 4 | **완성감=공유 유발** | 표지·완성작은 자랑하고 싶게. 공유가 바이럴 트리거 |
| 5 | **신뢰 신호** | 마켓엔 평점·미리보기·AI 라벨. 정산은 투명 명세 |
| 6 | **모바일 우선** | 핵심 CTA는 하단 고정 바. 2단 레이아웃은 세로 스택 전환 |

## 1. 글로벌 (공통 시스템)

### 1-1. 정보 구조 / 내비게이션
```
글로벌 네비:  플롯위버(로고)   [창작]  [마켓]            [🔔] [지갑] [프로필▾]
                                                         (비로그인 시 → [로그인][무료로 시작])
```
- **2개 핵심 영역**: `창작`(스튜디오: 위저드→에디터→표지→작업실) · `마켓`(탐색→판매페이지→뷰어).
- 프로필 메뉴: 내 작업실 / 구매내역·서재 / 지갑(크레딧·잉크) / 창작자 정산 / 설정 / 로그아웃.

### 1-2. 반응형 그리드
- **Desktop** ≥1024px: 12컬럼. 2단 레이아웃 사용(위저드+미리보기, 관계도, 표지, 정산).
- **Tablet** 768~1023px: 2단 → 1단 우선 축소.
- **Mobile** <768px: 1단 세로 스택. **핵심 CTA(생성·구매·후원·결제)는 하단 고정 바.** 위저드는 "한 그룹 = 한 스크롤".

### 1-3. 공통 컴포넌트 (재사용)
| 컴포넌트 | 설명 |
| --- | --- |
| **하이브리드 입력** | `▼Select(추천값)` + 마지막 항목 `+ 직접입력`. 모든 설정 입력의 기본형(`02`·`03`) |
| **작품 카드** | 🖼표지 + 제목 + (작가) + ★평점 + 가격/무료/후원 배지. 캐러셀·그리드 공용 |
| **AI 생성 배지** | "AI 생성" / "AI 보조" 라벨(기여 등급, `19`·`20`). 카드·판매페이지·뷰어에 표기 |
| **크레딧/잉크 칩** | 잔액 표시 + [충전]. 생성·구매·후원 직전 노출 |
| **스텝 인디케이터** | 위저드 4스텝 진행 표시 |
| **진행률 바** | 생성 로딩·미성년 한도 사용량 |
| **하단 고정 CTA 바** | 모바일 전역 주 액션 |

### 1-4. 공통 화면 상태 (모든 화면 설계 시 반드시 포함)
- **빈 상태(Empty)**: 첫 진입(작품 0개 등) — 안내 일러스트 + 주 CTA.
- **로딩(Loading)**: 스켈레톤 카드 / 진행률.
- **에러(Error)**: 원인 + 재시도 버튼. 생성 실패는 "크레딧 미차감" 안심 문구(`12`).
- **성공(Success)**: 토스트/체크. 생성 완료·구매 완료·확정 등.
- **권한·게이트**: 비로그인→로그인 유도 / 19금→성인 인증 / 미성년→한도·차단.

## 2. 화면 명세 (MVP 핵심 흐름 우선)

> 각 화면: **라우트 · 목적 · 우선순위 · 레이아웃 · 주요 컴포넌트 · 콘텐츠 · 상태 · 인터랙션**

### 2-1. 랜딩 (L) · `/` · **P0**
**목적**: "이거 진짜 되네?" 호기심 유발 → 가입 전환.
```
┌──────────────────────────────────────────────────────────┐
│ 플롯위버   [창작]  [마켓]            [로그인] [무료로 시작] │
├──────────────────────────────────────────────────────────┤
│        설정만 넣으면, AI가 웹소설을 완성합니다              │ ← 히어로 카피
│        회차별 본문 + 책 표지까지 자동으로                  │
│              [ ▶ 무료로 시작하기 ]                         │ ← 주 CTA
├──────────────────────────────────────────────────────────┤
│  🔥 지금 인기 작품                          < · · · · >    │ ← 작품 카드 캐러셀
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌────┐                       │
├──────────────────────────────────────────────────────────┤
│  작동 원리   ①설정입력 → ②AI생성 → ③표지·판매            │
│  예시 결과물 갤러리 | 요금/수익 안내 | 푸터                │
└──────────────────────────────────────────────────────────┘
```
- **컴포넌트**: 히어로(카피+주CTA), 인기작 캐러셀(작품 카드), 3단계 설명, 예시 갤러리, 요금 안내, 푸터.
- **상태**: 인기작 로딩(스켈레톤) / 데이터 0건 시 예시 더미.
- **인터랙션**: 주 CTA → 가입. 작품 카드 → 판매페이지(M2).

### 2-2. 로그인·회원가입 (A) · `/auth` · **P0**
**목적**: 마찰 최소 가입(소셜 우선).
- **컴포넌트**: 소셜 로그인 버튼(카카오/구글/애플), 이메일 옵션, [약관 동의] 진입.
- **분기 화면**(가입 필수 단계): **약관 동의(A1)** → (만14세 미만) **법정대리인 동의(A2)** → 온보딩.
- **성인 인증 게이트(AV)**: 19금 생성·열람·판매·구매 직전 공통 모듈(PASS 등). 비성인/미성년 → 차단.
- **상태**: 동의 미체크 시 [계속] 비활성 / 인증 실패·미성년 안내.

### 2-3. 설정 위저드 (C1) · `/create` · **P0 ★핵심**
**목적**: 막막함 없이 4스텝으로 작품 설정 완성(`02`·`03`).
```
┌──────────────────────────────────────────────────────────┐
│ ← 뒤로     새 작품 만들기              [임시저장]          │
├──────────────────────────────────────────────────────────┤
│  ①기본설정 ─── ②서사설정 ─── ③관계도 ─── ④출력설정       │ ← 스텝 인디케이터
│   ●━━━━━━━━━━○──────────○──────────○                      │
├──────────────────────────────────────────────────────────┤
│  [그룹1] 기본 설정                                         │
│  배경 시대   ▼ 중세 유럽            (+ 직접입력)           │
│  장르(복수)  [회귀x][복수x][+추가]  (+ 직접입력)           │
│  등장인물                                  [+ 인물 추가]   │
│  ┌──────────────────────────────────────────────┐         │
│  │ 이름____ 외형▼ 신체특징▼ 성격▼ 주인공[(•)예( )아니오]│ │ ← 인물 카드(반복)
│  └──────────────────────────────────────────────┘         │
│                              [ 다음: 서사설정 → ]          │
└──────────────────────────────────────────────────────────┘
```
- **4그룹**: ①기본(시대·장르·인물) ②서사(`목표▼` `갈등▼` `흐름____(+AI추천)` `결말▼`) ③관계도(C2) ④출력(`시점▼` `회차수▼` `분량▼` `등급▼` `제목____(+AI추천)`).
- **컴포넌트**: 스텝 인디케이터, 하이브리드 입력 다수, 반복 추가형 **인물 카드**, [임시저장], [AI 추천 자동채움].
- **상태**: 필수 누락 시 다음 버튼 비활성+필드 하이라이트(`12` VALIDATION) / 임시저장 토스트 / "한 줄 시놉시스→4단 자동제안" 로딩.
- **인터랙션**: 그룹 간 이동, 인물 추가/삭제, 직접입력 토글. 모바일: 한 그룹=한 화면 + 하단 [다음].

### 2-4. 관계도 빌더 (C2) · `/create/relations` · **P0 ★차별화**
**목적**: 인물 관계와 **회차별 변화**를 시각화(일관성 엔진의 입력).
```
┌───────────────────────────────────┬──────────────────────┐
│   관계 그래프 캔버스                │  관계 편집 패널       │
│        (카엘)──연인──▶(리나)       │  From ▼ 카엘          │
│          │                         │  To   ▼ 리나          │
│         원수                       │  관계  ▼ 연인 (+입력) │
│          ▼                         │  방향  (•)양방향      │
│        (악역)                      │  ─ 관계 변화 ───────   │
│   [+ 관계 추가]                    │  5화: 동료→연인        │
│                                    │  12화: 연인→원수 [+]   │
├───────────────────────────────────┴──────────────────────┤
│  ← 이전              [ ✦ 1회차 생성하기 ]                  │ ← 생성 트리거
└───────────────────────────────────────────────────────────┘
```
- **컴포넌트**: 노드-엣지 그래프 캔버스, 우측 편집 패널, **관계 변화 타임라인**(회차별), [자동 추천], 주 CTA [1회차 생성].
- **상태**: 인물 2명 미만 시 안내 / 자동추천 로딩 / 빈 캔버스 가이드.
- **인터랙션**: 노드 드래그, 엣지 생성, 라벨 선택, 타임라인 항목 추가. 모바일: 그래프 위/편집 아래 스택.

### 2-5. 생성 로딩 (C3) · `/create/generating` · **P0**
**목적**: 가장 이탈 큰 구간 — 불안 해소.
```
┌──────────────────────────────────────────────────────────┐
│            ✦ AI가 1회차를 집필 중입니다...                 │
│         [██████████░░░░░░░░]  62%   예상 20초             │ ← 진행률·예상시간
│   ┌── 입력 요약 ──────────────────────┐                   │
│   │ 중세 유럽 · 회귀/복수 · 카엘(주인공)│                   │
│   │ 목표: 복수 / 결말: 복수 완성        │                   │
│   └────────────────────────────────────┘                  │
│   💡 Tip: 생성 후 마음에 안 드는 부분은 재생성할 수 있어요 │
└──────────────────────────────────────────────────────────┘
```
- **컴포넌트**: 진행률 바+예상시간, 설정 요약 카드, 회전 팁.
- **상태**: 진행/완료(→C4 전환)/**실패(재시도 + "크레딧 미차감")**.

### 2-6. 에디터 / 뷰어 (C4) · `/work/:id` · **P0 ★Aha**
**목적**: "내 설정이 소설이 됐다" 감동 + 편집·이어쓰기(기여 = `20`).
```
┌──────────────────────────────────────────────────────────┐
│ ← 작업실   「제목」                  [표지 만들기][판매등록]│
├──────────┬───────────────────────────────────────────────┤
│ 회차 ≡    │  1화. 회귀                                     │
│ ●1화 회귀 │  눈을 떴을 때, 나는 10년 전으로 돌아와 있었다. │ ← 본문 뷰어/에디터
│  2화 ...  │  ...                                          │
│  [+ 생성] │  ─────────────────────────────────────         │
│           │  [✎ 부분 수정] [↻ 이 회차 재생성]              │
│           │              [ 다음 회차 생성 → ]              │
└──────────┴───────────────────────────────────────────────┘
```
- **컴포넌트**: 좌측 회차 목록(≡), 본문 뷰어/인라인 에디터, [부분 수정][재생성][다음 회차 생성], 상단 [표지 만들기][판매등록].
- **기여 게이지(`20`)**: 판매 등록 전 "직접 편집" 유도용 진행 게이지(편집 비중) — 에디터 내 노출.
- **상태**: 읽기/편집 모드 토글, 재생성 로딩, 저장 토스트, 마지막 회차일 때 [다음 생성] 강조.
- **인터랙션**: 회차 전환, 문단 선택→부분 수정, 직접 타이핑(내 문장 추가). 모바일: 회차 목록 드로어.

### 2-7. 표지 생성 (C5) · `/work/:id/cover` · **P0 ★바이럴**
**목적**: 시각적 완성감 → 공유 유발.
```
┌───────────────────────────────┬──────────────────────────┐
│  스타일 ▼ 웹툰풍               │   표지 후보               │

│  [ ✦ 표지 생성 ] [ ↻ 다시 ]   │                          │
├───────────────────────────────┴──────────────────────────┤
│                          [ 이 표지로 확정 ]               │
└───────────────────────────────────────────────────────────┘
```
- **컴포넌트**: 좌측 스타일 옵션(스타일·색감·제목·인물포함), 우측 후보 그리드(선택), [생성][다시][확정].
- **상태**: 생성 로딩(후보 스켈레톤)/선택/확정/실패. 확정 후 [공유] 노출(바이럴).

### 2-8. 내 작업실 / 라이브러리 (D1) · `/library` · **P0**

- **상태**: **빈 상태(첫 사용자)** = "첫 작품을 만들어보세요" + 주 CTA / 로딩 스켈레톤.

### 2-9. 마켓 홈·탐색 (M1) · `/market` · **P1**
**목적**: 구매 진입 — 발견.
```
┌──────────────────────────────────────────────────────────┐
```
