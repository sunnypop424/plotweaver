## 1. 저작권 · 표절 대응

**핵심 목표**: 타 작품 표절·무단 도용을 차단하고, AI 생성물의 권리 귀속을 명확히 한다.

| 단계 | 대응 방안 |
| --- | --- |
| **예방** | • 가입·등록 시 **창작 윤리 약관** 동의(타 저작물 무단 사용 금지)<br>• 레퍼런스 입력란에 "특정 작가·작품 문장 복제 금지" 가이드 고지<br>• AI 프롬프트에 표절 방지 가드레일(원문 인용·고유명사 도용 차단) 적용<br>• **창작 기여 게이트(F-16, `20`)**: 유료 판매는 회원의 편집·증감으로 기여 등급 `ai_assisted` 이상일 때만 가능 → 무편집 AI 생성물의 마켓 유입 차단 |
| **탐지** | • 판매 등록 시 **유사도 검사**(기존 등록작·외부 DB 대조, 임계치 초과 시 보류)<br>• 사용자 **신고 버튼**(표절 의심 작품 신고)<br>• **편집 이력·기여 기록**(F-16)으로 창작 기여 실재 여부 확인 |
| **대응** | • 표절 확정 시 작품 **비공개·판매 중단·정산 보류**<br>• 반복 위반자 **이용 정지·수익 환수**<br>• 권리 귀속: 약관에 "생성물의 **이용권**은 창작자, AI 저작물성은 불확실·창작 기여로 보강" 명시(`07` B-1, `19` §2) |

**정책 문구 예시**: "본 서비스로 생성된 콘텐츠의 상업적 이용권은 창작자에게 귀속되나, 타인의 저작권을 침해하는 입력·산출물에 대한 책임은 창작자에게 있습니다."

> 🔗 **창작 기여 = 저작권 리스크의 핵심 방어선**: 순수 AI 생성물은 저작물성이 약해(`19` §1-2) "팔 권리"의 전제가 흔들린다. F-16(`20`)이 **편집·선택·증감을 유도→측정→게이트→증거화**하여 이 리스크를 정면 대응한다. 저작권 귀속의 상세 정책은 `19`, 시스템 설계는 `20` 참조.

## 2. 콘텐츠 심의 (불법·유해 콘텐츠)

**핵심 목표**: 불법(아동 성착취물, 혐오·폭력 조장 등) 및 등급 위반 콘텐츠의 생성·유통을 차단한다.

| 단계 | 대응 방안 |
| --- | --- |
| **예방** | • **생성 단계 필터**: 금지 키워드/주제 입력 차단(불법·미성년 성적 묘사 등)<br>• 연령 등급(필수 입력) 기반 표현 수위 자동 제한<br>• 19금 콘텐츠는 **성인 인증** 사용자만 생성·열람 |
| **탐지** | • 생성 결과 **자동 모더레이션**(유해 표현 스코어링) → 임계치 초과 시 차단/플래그<br>• 판매 등록 전 **사전 검수 큐**(고위험 등급은 사람 검수 병행)<br>• 신고 접수 시스템 |
| **대응** | • 위반물 **즉시 비공개·삭제**, 작성자 제재<br>• 불법물은 **수사기관 신고 의무** 이행<br>• 단계적 제재 정책(경고→정지→영구차단) |

> **운영 체계**: 자동 필터(1차) + 사람 검수(2차, 19금/신고건) + 신고 대응(3차) 3단 방어선.

## 3. 정산 투명성 · 분쟁

**핵심 목표**: 창작자가 매출·수수료·정산을 신뢰할 수 있게 하고 분쟁을 예방한다.

| 단계 | 대응 방안 |
| --- | --- |
| **예방** | • **수수료·정산 비율 사전 명시**(30%, Pro 20% 등) 및 약관 동의<br>• 판매자 대시보드에 **실시간 매출·차감 내역** 공개<br>• 정산 주기·최소 출금액·세금 처리 사전 안내 |
| **탐지** | • 거래 원장(ledger) 기록·대사(reconciliation) 자동화<br>• 정산 오류·누락 모니터링 알림 |
| **대응** | • **정산 이의제기 채널** 운영(영업일 N일 내 회신 SLA)<br>• 환불 발생 시 정산 차감 규칙 명문화<br>• 분쟁 시 거래 로그 기반 객관적 중재 |

---

## 4. 부정거래 (어뷰징) 방지

**핵심 목표**: 자전거래·환불 악용·랭킹 조작 등 마켓 신뢰를 해치는 행위를 차단한다.

| 유형 | 탐지·대응 방안 |
| --- | --- |
| **자전거래(셀프구매)** | 동일/연관 계정·결제수단·기기·IP 간 거래 탐지 → 정산 제외·계정 제재 |
| **환불 어뷰징** | 열람 완료분 환불 제한, 환불율 이상 계정 모니터링, 환불 정책 명문화(미열람분 한정 등) |
| **랭킹·리뷰 조작** | 어뷰징 의심 리뷰·조회 필터링, 구매자 한정 리뷰 작성 |
| **대량 어뷰징** | 이상거래 탐지(FDS) 룰셋 + 임계 초과 시 자동 홀드 → 수동 검토 |

| 단계 | 대응 방안 |
| --- | --- |
| **예방** | 결제 본인확인, 환불 정책 명시, 신규 판매자 한도 |
| **탐지** | FDS(이상금융거래탐지) 룰·패턴 분석, 어뷰징 지표 대시보드 |
| **대응** | 정산 보류·계정 정지·수익 환수, 반복 시 영구 차단 |

## 5. AI 생성물 고지 · 책임

**핵심 목표**: 소비자가 AI 생성 콘텐츠임을 인지하게 하고, 산출물 품질·오류에 대한 책임 범위를 명확히 한다.

| 단계 | 대응 방안 |
| --- | --- |
| **예방** | • 모든 작품에 **기여 등급별 AI 라벨** 표기(F-16: "AI 생성"/"AI 보조"/"사용자 주도", 판매페이지·뷰어)<br>• 약관에 산출물 면책·창작자 검수 책임 명시(`07` 제10조) |
| **탐지** | • 라벨 누락·조작 작품 점검, AI 산출 메타데이터·**기여 등급 기록**(F-16) |
| **대응** | • 허위·기만적 표기·기여 조작 시 제재<br>• 산출물 오류(사실 오인·부적절 표현)는 창작자 수정 책임, 플랫폼은 도구 제공자로서 면책 범위 고지 |

> 라벨은 회원이 임의 표기하지 않고 **창작 기여 등급으로 자동 결정**(F-16, `19` §5)되어 표시의 객관성·기만 방지를 확보한다.

> **소비자 보호**: "이 작품은 AI로 생성되었습니다" 고지 + 미리보기 제공으로 구매 전 판단 가능하게.

## 6. 개인정보 · 미성년자 보호 *(추가 식별 리스크)*

**핵심 목표**: 10대 타겟이 포함된 만큼 개인정보·미성년 보호 규제를 준수한다.

| 단계 | 대응 방안 |
| --- | --- |
| **예방** | • 개인정보 최소 수집·이용 동의(개인정보보호법 준수)<br>• 만 14세 미만 **법정대리인 동의** 절차<br>• 미성년 계정은 19금 콘텐츠 생성·열람·결제 한도 제한 |
| **탐지** | • 연령 인증·결제 본인확인으로 미성년 성인콘텐츠 접근 차단<br>• 비정상 결제(미성년 고액) 모니터링 |
| **대응** | • 미성년 결제 취소·환불 절차 운영<br>• 개인정보 유출 시 신고·통지 의무 이행 |

## 7. 통합 운영 체계 (Trust & Safety)

```
[1차 자동 방어]  생성 필터 · 모더레이션 · 유사도 검사 · FDS
        ↓ (플래그/임계 초과)
[2차 사람 검수]  19금·신고·고위험 등록건 수동 리뷰
        ↓ (위반 확정)
[3차 제재·대응]  비공개·삭제·정산보류·계정정지·수사기관 신고
        ↺
[피드백]  신고·이의제기 채널 → 룰셋·약관 개선
```

| 항목 | 담당(R&R) |
| --- | --- |
| 정책 수립·약관 | PM · 법무 |
| 자동 필터·FDS 구축 | 개발 · 데이터 |
| 사람 검수·신고 처리 | 운영(Trust&Safety) |
| 정산·환불·분쟁 | 재무 · CS |

| 단계 | 적용 리스크 대응 |
