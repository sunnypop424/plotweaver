# 기능 명세서 (개발 착수용 PRD)

> MVP 개발 착수를 위한 기능 단위 명세. 각 기능 = ID·사용자스토리·상세동작·입출력·검증·완료조건(AC)·의존성·우선순위.
> 범위: Phase 1(MVP) 중심 + Phase 2(마켓·후원) 주요 기능.
> 작성: 수석 PM / 최종수정: 2026-06-26

---

## 0. 표기 규칙
- **우선순위**: P0(MVP 필수) / P1(마켓 오픈) / P2(이후)
- **AC** = Acceptance Criteria(완료 조건). 이 조건을 만족해야 '완료'로 간주.
- 화면 ID는 `04`·`10` 문서와 연동(C1, M2, P1 등).

## 2. 기능 명세 — 인증/온보딩

### F-01 회원가입·로그인 `P0`
- **스토리**: 사용자로서, 빠르게 가입해 바로 창작을 시작하고 싶다.
- **동작**: 소셜/이메일 인증 → 약관 동의(F-02) → 가입 완료.
- **AC**:
  - [ ] 소셜 3종·이메일로 가입/로그인 가능
  - [ ] 미인증 사용자는 생성·결제 기능 접근 차단
- **의존**: F-02

### F-02 약관 동의 & 연령 확인 `P0`
- **동작**: 필수(이용약관/개인정보/거래정책)·선택(마케팅) 분리 동의. '만 14세 미만' 선택 시 법정대리인 동의(F-03) 분기.
- **입력**: 약관 동의 체크, 연령 자가확인
- **AC**:
  - [ ] 필수 항목 전체 동의 시에만 가입 진행
  - [ ] 전체동의는 보조(강제 선택 아님)
  - [ ] 약관 변경 시 재로그인 시 재동의 모달
- **의존**: `07` 약관

### F-03 법정대리인 동의 `P1`
- **동작**: 만 14세 미만 보호자 본인확인 → 가입 승인. 미성년 이용 제한 고지.
- **AC**: [ ] 보호자 인증 완료 전 미성년 가입 미완료

### F-04 성인 인증(19금 게이트) `P1`
- **스토리**: 19금 콘텐츠 생성/열람/판매 시 본인 연령을 인증한다.
- **동작**: 트리거(위저드 19세 등급 / M2·M3 열람 / S2 판매등록) → 미인증 시 외부 인증(PASS 등) → 성공 시 계정 플래그 저장, 실패 시 차단.
- **AC**:
  - [ ] 인증된 성인만 19금 액션 가능
  - [ ] 1회 인증 후 재요청 없음
  - [ ] 인증 결과(성인 여부)만 저장, 원본 최소 보관
- **의존**: F-01

## 3. 기능 명세 — 창작(생성)

### F-10 설정 입력 위저드 `P0` ★핵심
- **스토리**: 창작자로서, 설정만 단계별로 입력해 소설 생성을 준비하고 싶다.
- **동작**: 4그룹(기본/서사/관계/출력) 스텝 입력. 모든 필드 Select+직접입력. AI 추천값 자동 채움. 임시저장.
- **입력(필수)**: 배경시대, 등장인물(1+, 주인공 1), 장르(1+), 목표·동기, 핵심갈등, 스토리흐름, 결말방향, 화자시점, 회차·분량, 연령등급, 제목(또는 AI자동)
- **검증**: 필수 체크리스트(`02`) 충족 시에만 '생성하기' 활성화
- **AC**:
  - [ ] 각 필드 Select/직접입력 모두 동작
  - [ ] 인물 카드 동적 추가/삭제
  - [ ] 필수 누락 시 생성 버튼 비활성 + 안내
  - [ ] 임시저장 후 이어쓰기 가능
- **의존**: F-11, F-12

### F-11 인물 관계도 빌더 `P0`
- **스토리**: 인물 간 관계와 그 변화를 설정해 일관성을 확보하고 싶다.
- **동작**: 인물=노드, 관계=엣지 그래프. 관계 라벨(Select+입력)·방향 설정. **회차/단계별 관계 변화** 타임라인 추가. 장르 기반 자동 추천.
- **데이터**: `relations[{from,to,timeline[{stage,label}]}]` (`02` 스키마)
- **AC**:
  - [ ] 2+ 인물 간 관계 생성/수정/삭제
  - [ ] 관계 변화(단계별) 입력·저장
  - [ ] 생성 시 관계 데이터가 프롬프트에 반영
- **의존**: F-10(인물 데이터)

### F-12 회차별 본문 자동 생성 `P0` ★핵심
- **스토리**: 입력 설정으로 회차 본문을 자동 생성받고 싶다.
- **동작**: 설정+관계도 → AI 프롬프트 구성 → 비동기 생성 → 진행률 표시 → 결과 반환. 1회차씩/일괄 선택. **이전 회차 맥락을 누적**해 일관성 유지.
- **입력**: 작품 설정 일체 / **출력**: 회차 본문(지정 분량)
- **검증**: 크레딧 잔액 확인 후 차감(F-30)
- **AC**:
  - [ ] 설정 기반 회차 본문 생성·저장
  - [ ] 이전 회차/관계 변화가 다음 회차에 반영(일관성)
  - [ ] 생성 실패 시 크레딧 미차감·재시도
  - [ ] 생성 원가/토큰 로깅
- **의존**: F-10, F-11, F-30

### F-13 회차 재생성·부분 수정 `P0`
- **동작**: 회차 전체 재생성 또는 선택 문단 수정/톤 조정. 직접 편집 가능.
- **AC**: [ ] 재생성 시 크레딧 정책 적용 [ ] 수동 편집 저장

### F-14 AI 책 표지 생성 `P0` ★핵심
- **스토리**: 작품에 어울리는 표지를 자동 생성받고 싶다.
- **동작**: 스타일/색감/제목/인물포함 선택 → 후보 N개 생성 → 선택·확정. 재생성 지원.
- **입력**: 제목·장르·분위기·스타일 / **출력**: 표지 이미지 후보
- **AC**:
  - [ ] 스타일별 표지 생성·재생성
  - [ ] 후보 중 선택→작품 적용
  - [ ] Free 등급은 워터마크
- **의존**: F-30

### F-15 작품 보관함(라이브러리) `P0`
- **동작**: 생성 작품/회차 저장·목록·상태(비공개/공개/판매중) 관리·이어쓰기.
- **AC**: [ ] 작품 CRUD [ ] 상태 필터 [ ] 재진입 편집

### F-16 창작 기여 증명·편집 게이트 `P1` ★저작권 전제
- **스토리**: AI 생성물을 마켓에 팔려면 내 창작 기여가 필요하다 — 그 기여를 유도·측정·증명받고 싶다.
- **배경**: "설정 입력만"으로는 저작물성이 약할 수 있어(`19` §1-2) **유료 판매의 법적 전제**가 흔들림. 편집을 "권장"이 아니라 **게이트+증거 시스템**으로 강제(`20`).
- **동작**: ①에디터 편집 유도(기여도 게이지) ②편집·선택·증감 자동 측정 → 기여도 점수·등급(ai_generated/ai_assisted/user_led) ③**유료 판매 등록 시 `ai_assisted` 이상 게이트** ④편집 이력·기여 내역서 보존(append-only).
- **입력**: 에디터 편집 행위 / **출력**: 기여 등급·라벨, 판매 가능 여부, 창작 기여 내역서
- **AC**:
  - [ ] AI 초안(v0)~최종(vN) revision 기록, 편집 이벤트 append-only 로깅
  - [ ] 기여도·등급 **서버 재계산**(클라값 불신), 사소변경(오탈자·공백) 배제
  - [ ] 유료 판매 시 ai_generated 회차 포함이면 등록 차단(`CONTRIBUTION_REQUIRED`)
  - [ ] 등급→AI 라벨 자동 결정(`19` §5)
  - [ ] 창작 기여 내역서 내보내기(PDF/JSON)
- **의존**: F-12(초안 생성=기준선), F-13(편집), F-20(판매 등록)
- **상세**: `20_창작기여증명_및_편집게이트.md`
